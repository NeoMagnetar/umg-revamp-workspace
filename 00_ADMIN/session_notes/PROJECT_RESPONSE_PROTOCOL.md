# Project Response Protocol

## Standard Response Intent
Responses in this project should aim to do four things:
1. move the work forward
2. preserve organizational clarity
3. update live tracking when relevant
4. produce artifacts when a section becomes coherent enough to freeze

## Preferred Response Elements
When relevant, include:
- active stack snapshot
- envoy intuition
- current context / MOLT map
- formal response content
- file or zip handoff if artifacts were produced
- PowerShell import/sort block when files were generated
- metadata block

## Important Clarification
A zip or PowerShell block is most useful when files are actually produced.
It is not required for replies that are purely conversational or diagnostic.
