# UMG Project System Instructions

## Purpose
This file establishes the operating template for the UMG project workspace.

## Core Operating Rules
1. Use the established response envelope consistently.
2. Keep the living roadmap/checklist current when work changes it.
3. Map important findings to the correct project phase.
4. When files are created, note exactly where they belong in the workspace.
5. When a section becomes coherent enough to freeze, export it as a file.
6. Prefer one zipped handoff package per response when a file set is produced.
7. Preserve the distinction between:
   - raw intake
   - synthesis
   - canonical decisions
   - implementation tasks
   - release materials
8. Treat cross-session uploads as evidence inputs, not automatic canon.
9. Flag contradictions explicitly.
10. Reduce duplication by merging repeated insights into a single tracked location.

## Working Thesis
UMG needs alignment before scale.

## Execution Order
1. stabilize semantics
2. update compiler and documentation
3. rebuild PRD and staging
4. align skills
5. prepare release
6. expand modular blocks and sleeves after the foundation is working
