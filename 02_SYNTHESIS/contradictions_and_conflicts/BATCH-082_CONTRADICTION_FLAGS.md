# BATCH-082 — Contradiction / Ambiguity Flags

## Contradiction Watch
HIGH

## Main Flags
### 1. Template vs Primary vs Blueprint vs Goal
The batch uses overlapping top-level names that are not yet fully stabilized.

### 2. Merge Expansion vs Older Merge Definitions
Earlier documents often frame Merge as support information or logic fusion. This batch broadens merge into a more general cognitive fusion operator that may involve any block types.

### 3. Instruction Flexibility
Instruction blocks are described both as mergeable and as lower-hierarchy attachments. This flexibility is useful, but not formalized.

### 4. Hierarchy vs Peer Authority
The batch wants both strict inheritance and equal-authority peer structures. The boundary between those structures remains unresolved.

### 5. Builder Grammar vs Runtime Canon
The batch clarifies a user-facing grammar, but the relationship between that grammar and the earlier runtime priority/token-weight model is not fully reconciled.

### 6. Color / Legend Instability
Color mapping and visual semantics were revised multiple times, so UI legend choices remain provisional.

## Intake Conclusion
Use this batch for:
- role-language harmonization
- builder PRD decisions
- hierarchy / skeleton design
- merge-law comparison work

Do not use it alone to settle:
- final top-level role naming
- final merge law
- final priority model
- final export/runtime structure
