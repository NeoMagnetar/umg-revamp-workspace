# BATCH-178 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** CEL canonical status  
**Tension:** CEL is used functionally as a modular layer, but its canonical status is not established.  
**Why it matters:** Later docs should not silently canonize it from this source alone.

## Flag 2
**Topic:** Chaos Lens ontology  
**Tension:** Chaos Lens behaves like an overlay or mode, but its formal taxonomy is unresolved.  
**Why it matters:** Overlay law and lens language require clearer boundaries.

## Flag 3
**Topic:** Speculative escalation  
**Tension:** The farmer evolution sequence starts grounded and later becomes increasingly speculative.  
**Why it matters:** Canon quality depends on separating durable runtime patterns from imaginative overextension.

## Flag 4
**Topic:** Stopping conditions  
**Tension:** Recursive self-evolution is described, but no stopping rules are defined.  
**Why it matters:** Safe bounded self-modification needs explicit termination and acceptance criteria.

## Flag 5
**Topic:** Missing formal runtime schema  
**Tension:** Governance, scoring, and merge behavior are implied, but no exact data model or validation spec is finalized.  
**Why it matters:** Later compiler/runtime work must normalize these surfaces before implementation.
