# BATCH-175 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** Stack terminology overload  
**Tension:** The source repeatedly distinguishes strict same-type compiler stack from broader everyday layered grouping, but the language remains strained.  
**Why it matters:** This is one of the most important unresolved naming tensions in the system.

## Flag 2
**Topic:** Domain/profile/bundle naming  
**Tension:** Domain is increasingly favored, but profile and bundle remain nearby alternatives and the human-facing grouping term is not fully final here.  
**Why it matters:** Public docs and UI naming should not freeze too early around a still-contested term.

## Flag 3
**Topic:** Candidate canon versus promoted canon  
**Tension:** The source contains very strong canon-style prose, but not every sentence is guaranteed already-promoted law.  
**Why it matters:** Later control-room synthesis must distinguish “locked in chat” from “canonically ratified in project records.”

## Flag 4
**Topic:** Trigger and merge maturity  
**Tension:** Trigger v0 and merge rules are repeatedly specified, but later implementation-oriented batches may refine or partially diverge from these formulations.  
**Why it matters:** Compiler work should normalize against the latest approved surfaces, not just this source in isolation.

## Flag 5
**Topic:** Documentation order as governance  
**Tension:** The source strongly treats document order as a drift-prevention mechanism.  
**Why it matters:** Later teams must decide exactly how rigidly sequence is enforced outside authoring sessions.
