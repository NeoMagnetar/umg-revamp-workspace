# BATCH-126 — Contradiction Flags

## Scope Note
These are contradiction or tension flags, not canon rulings.

## Flag 1 — MOLT as Authority vs MOLT as Semantic Ordering
This source frames MOLT as the fixed hierarchy of authority and uses language such as “MOLT decides who is allowed to win.”
Other evidence sources frame MOLT primarily as semantic ordering while governance performs override and suppression.
This needs direct synthesis review.

## Flag 2 — Governance Optionality vs Governance as Sole Override Layer
This batch treats governance as useful but not strictly required.
Other evidence frames governance as the only true override mechanism.
This creates ambiguity around what happens in conflict scenarios when governance is absent.

## Flag 3 — NeoStack Default Compose Mode
The source preserves unresolved tension over whether NeoStack default should be bundle or merge.
This affects both compiler defaults and Studio UX.

## Flag 4 — Terminology Stability Around Column / cocoon
The structural role is relatively clear, but user-facing terminology remains somewhat fluid between Column, MOLT Column, and cocoon.
This is a documentation and builder-UX normalization issue.

## Flag 5 — Trigger Complexity / Pass Depth
The source recognizes trigger logic as a determinism risk area and suggests minimal v0 complexity, but does not lock final pass behavior.
This remains partly open for compiler staging.

## Flag 6 — Governance Object Semantics
Capstone, crown, overlay, scoped object, off/defer/apply modes, and subtree behavior were discussed but not fully frozen here.
This needs later formalization before implementation hardening.
