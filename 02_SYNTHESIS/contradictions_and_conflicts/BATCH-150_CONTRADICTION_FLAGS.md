# BATCH-150 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** Template versus canon system  
**Tension:** The chat uses “template” language heavily, but it does not sharply distinguish template convenience from canon structural law.  
**Why it matters:** Reusable patterns should not be mistaken for finalized ontology without review.

## Flag 2
**Topic:** Implicit NeoStack language  
**Tension:** The layered ordering is strong, but NeoStack terminology is only implied rather than explicitly used.  
**Why it matters:** Later documentation should make clear whether this is merely stack-like or truly promoted into NeoStack canon.

## Flag 3
**Topic:** Governance duplication  
**Tension:** Governance is embedded separately inside each agent through alignment and instruction layers rather than centralized.  
**Why it matters:** This may be practical for templates but could conflict with later centralized governance models.

## Flag 4
**Topic:** Trigger simplicity  
**Tension:** `trigger_chain_map` is useful, but may be too lightweight for more complex runtime orchestration.  
**Why it matters:** Future runtime design may need a richer trigger model.

## Flag 5
**Topic:** Public abstraction mapping gap  
**Tension:** The need for a simplified public builder is clear, but the chat does not fully specify how each public-facing choice maps to the internal block grammar.  
**Why it matters:** Product simplicity should not come at the cost of ambiguous schema translation.
