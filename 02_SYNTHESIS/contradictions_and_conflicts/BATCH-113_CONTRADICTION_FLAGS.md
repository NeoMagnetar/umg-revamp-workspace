# BATCH-113 — Contradiction Flags

## High-value contradictions and unresolved tensions

### 1. Archive schema as tool vs canon
- The batch proposes a substantial schema, payload taxonomy, scope model, and conflict engine.
- It does not resolve whether those should become canonical project infrastructure or remain working archive tooling.

**Risk:** workflow design gets mistaken for doctrine.

### 2. Assistant-proposed structure vs project-ratified structure
- Much of the archive schema and workflow logic appears assistant-proposed in response to the user request.
- Separate explicit ratification is not shown.

**Risk:** over-promoting generated structure into canon.

### 3. Runtime-style sleeve JSON status
- A sleeve/runtime-style JSON appeared in responses.
- Its status as canon, example, or incidental template is unresolved.

**Risk:** accidental canonization of illustrative structure.

### 4. Automatic conflict resolution boundary
- Several auto-resolution methods are proposed.
- Acceptance criteria for when automation is allowed versus when manual review is required are not finalized.

**Risk:** hidden authority creep in archive tooling.

### 5. Scope and normalization rules remain partial
- Scope levels and conflict logic are proposed.
- Overlap rules, polarity rules, and normalization details are not fully specified.

**Risk:** inconsistent conflict emission across chats.

### 6. Trigger / priority / merge / bundle are archivable targets but underdeveloped here
- These areas are named and serialized, but not semantically advanced in this chat.

**Risk:** archive structure implies more semantic stability than the evidence supports.

## Recommended handling
- Keep this batch as evidence only.
- Use it to support archive tooling, evidence indexing, and conflict-workflow design.
- Require explicit project-level approval before treating the schema, scope model, or auto-resolution methods as stable infrastructure.
