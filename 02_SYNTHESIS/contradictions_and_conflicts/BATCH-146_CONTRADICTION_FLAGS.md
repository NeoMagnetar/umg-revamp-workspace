# BATCH-146 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** “Symbiotic with all systems”  
**Tension:** The phrase is persuasive and expansive, but no clear boundary is defined in the chat.  
**Why it matters:** Public language should remain legible without becoming overclaiming.

## Flag 2
**Topic:** Pseudo-spec fields as possible schema confusion  
**Tension:** Fields like `APP.ID`, `MODE`, `STACK`, and `FEATURES` looked schema-like but were ideation shorthand only.  
**Why it matters:** Presentation shorthand should not be mistaken for approved schema.

## Flag 3
**Topic:** Runtime metaphor versus canon vocabulary  
**Tension:** Vault, stack, graph, and agent were used productively, but without strict distinctions.  
**Why it matters:** Public docs may need a curated vocabulary layer to avoid concept drift.

## Flag 4
**Topic:** Broad domain coverage  
**Tension:** The chat covered barter, finance, farming, crypto, writing, memory, and more without prioritizing a first-release proof-of-value.  
**Why it matters:** Showcase breadth and roadmap commitment should remain separate.

## Flag 5
**Topic:** Safety/control named but unordered  
**Tension:** Fallback routing, override conditions, emergency flags, and alignment monitoring were all named, but no governance precedence order was defined here.  
**Why it matters:** Public demos may need simplified but explicit control logic.
