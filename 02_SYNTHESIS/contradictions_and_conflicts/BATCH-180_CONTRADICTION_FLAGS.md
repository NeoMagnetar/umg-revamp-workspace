# BATCH-180 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** Row versus category blockification  
**Tension:** The source repeatedly encountered confusion over whether a category should become one block, one per row, or one grouped summary.  
**Why it matters:** This is one of the highest-impact unresolved workflow ambiguities.

## Flag 2
**Topic:** Filename drift  
**Tension:** Naming drift repeatedly caused avoidable unpack failures.  
**Why it matters:** Stable filenames are part of the operational contract, not just convenience.

## Flag 3
**Topic:** Trustworthiness of prior artifacts  
**Tension:** Some earlier generated grouped files were explicitly rejected as malformed or under-granular.  
**Why it matters:** Not all prior artifacts in the chat should be treated as equally canonical references.

## Flag 4
**Topic:** MOLT consistency  
**Tension:** Similar materials were sometimes encoded under different MOLT types across domains.  
**Why it matters:** Later taxonomy and template canon work must normalize these assignments.

## Flag 5
**Topic:** Template canon still inferred  
**Tension:** The accepted block template was operationally established through repeated success, but not frozen in one explicit formal spec statement here.  
**Why it matters:** Formal canon may still need a single authoritative artifact-contract document.
