# BATCH-195 - Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
Topic: Role as a new layer
Tension: Role is now formalized separately from MOLT type, which may conflict with earlier UMG usage if not documented carefully.
Why it matters: Canon vocabulary can drift unless this split is made explicit everywhere.

## Flag 2
Topic: Stack dual use
Tension: Stack remains overloaded between compiler operation and organizational grouping.
Why it matters: New contributors may confuse mechanics with human-facing organization.

## Flag 3
Topic: Module and Composite Block / Macro status
Tension: These are useful distinctions but still partly helper-layer / authoring-layer rather than fully locked CSL terms.
Why it matters: Canon and convenience layers need cleaner separation.

## Flag 4
Topic: Old repo Merge language
Tension: The old repo README includes Merge as a canonical MOLT type, which conflicts with the current canon where Merge is an operation.
Why it matters: Migration cleanup is required to prevent backslide.

## Flag 5
Topic: Repo naming and implementation structure
Tension: `umg-core` and a greenfield compiler repo path are strongly recommended but not themselves canonized.
Why it matters: Repo strategy affects documentation, tooling, and contributor onboarding.
