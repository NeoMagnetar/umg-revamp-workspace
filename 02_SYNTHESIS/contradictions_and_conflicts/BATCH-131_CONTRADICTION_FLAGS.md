# BATCH-131 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** Same-MOLT merge restriction in v1  
**Tension:** Earlier explorations allowed or defended broader cross-MOLT merge concepts, while this batch simplifies Studio v1 to same-MOLT merge.  
**Why it matters:** Product simplification should not be mistaken for full semantic impossibility unless later canon explicitly says so.

## Flag 2
**Topic:** Governance scope narrowing  
**Tension:** This batch restricts v1 governance to NeoStack-level peer conflict and priority override, while other evidence explores broader governance capabilities.  
**Why it matters:** Studio v1 constraints and full UMG governance law must remain distinguished.

## Flag 3
**Topic:** Graph editing freedom  
**Tension:** The graph is tightly demoted here to overview/arrangement, while earlier app directions experimented with richer graph manipulation.  
**Why it matters:** UX implementation could silently reintroduce freeform graph semantics unless the boundary is enforced.

## Flag 4
**Topic:** Bundle result semantics  
**Tension:** This batch describes bundle as replacing originals with one composite result in Studio structure, which may differ from more compiler-theoretic descriptions where bundles preserve multiple items without new meaning.  
**Why it matters:** The project needs one stable explanation for compiler truth versus Studio-facing compressed representation.

## Flag 5
**Topic:** Repo/package shape  
**Tension:** Single-app clarity, minimal monorepo, and separated compiler-package strategies were all considered.  
**Why it matters:** Product staging should normalize one concrete repo structure before implementation drift compounds.

## Flag 6
**Topic:** Old visual language vs new architecture  
**Tension:** The user wanted continuity with the existing Studio feel while also structurally refactoring it.  
**Why it matters:** Ongoing UI translation work may create mismatches unless contracts stay explicit.
