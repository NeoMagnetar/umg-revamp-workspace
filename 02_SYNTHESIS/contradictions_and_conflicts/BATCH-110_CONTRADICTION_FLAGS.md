# BATCH-110 — Contradiction Flags

## Status
This batch contains important stabilizing material but also multiple intentionally unresolved areas that must not be silently promoted to canon.

## Major Tensions / Contradictions to Preserve
1. **Canon vs mapping language**
   - The source separates canon from academic-safe terminology mappings.
   - Risk: external-safe language gets mistaken for final project naming canon.

2. **Proof rigor vs human readability**
   - Formal semantics language was accurate but initially not intuitively legible.
   - Risk: rigorous docs become adoption-hostile without plain-language companions.

3. **MOLT order**
   - MOLT remained typed and important, but exact ordering/rigidity was explicitly marked as questionable.
   - Risk: future docs silently fix an order without project approval.

4. **Implicit merge vs explicit synthesis**
   - Preserved as open.
   - Risk: runtime or documentation assumes one behavior prematurely.

5. **NeoStack authority**
   - NeoStack exists as composed structure, but whether it introduces new authority remained unresolved.
   - Risk: stack composition semantics drift across docs and code.

6. **Priority semantics**
   - Priority was constrained to conflict sites in formal language, but its broader semantic weight remained open.
   - Risk: priority leaks into hidden authority logic.

7. **Executor interpretation**
   - The packet assumes executor-boundedness and specification/execution separation.
   - Risk: agent-facing explanations may still over-ascribe cognition to executors.

## Recommendation
Treat this batch as a **documentation- and semantics-hardening packet**, not as automatic canon. Use it to:
- tighten glossary wording,
- separate stable rules from translation layers,
- track unresolved issues explicitly,
- improve agent-safe documentation packaging.
