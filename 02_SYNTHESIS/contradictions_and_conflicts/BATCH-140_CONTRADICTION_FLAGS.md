# BATCH-140 — Contradiction Flags

## Scope Note
These are not canon decisions.
They are tensions or possible conflicts surfaced by the source.

## Flag 1
**Topic:** Conceptual merge versus executed merge  
**Tension:** Merge blocks are described conceptually, but not clearly executed in runtime code.  
**Why it matters:** Later documentation should distinguish design intent from implemented behavior.

## Flag 2
**Topic:** Memory trackers versus persistence  
**Tension:** Memory trackers are present in config, but not fully wired into persistence/runtime state.  
**Why it matters:** Runtime capability claims should remain scoped.

## Flag 3
**Topic:** Missing governance/safety formalization  
**Tension:** Control appears implicitly through values and tone rather than explicit governance blocks.  
**Why it matters:** A deployable UMG agent framework needs clearer bounded-agency structure than this prototype currently shows.

## Flag 4
**Topic:** Implicit MOLT mapping  
**Tension:** Functional layers are described in ways that resemble MOLT roles, but the source does not explicitly use canon MOLT labels everywhere.  
**Why it matters:** Mapping from prototype structure to formal canon should stay explicit.

## Flag 5
**Topic:** Pipeline ambition versus automation detail  
**Tension:** The source aims for “any bot of any kind” from structured input, but decision logic and cross-platform abstraction are not yet formalized.  
**Why it matters:** The prototype pattern is strong, but it should not be mistaken for a finished universal generator.
