# BATCH-142 — Canon Candidates

## Status Note
These are candidate canonical fragments extracted from a derived bot-maker taxonomy summary.
Inclusion here does not itself promote them to canon.

## Candidate Fragments
1. Identity is modular and composable rather than monolithic.
2. Both freeform labels and structured tags can exist, but only standardized tags should drive logic, filtering, and export.
3. Separate what a bot is like from how it thinks, how it speaks, what it can do, what it remembers, and what it is allowed to do.
4. Do not jump straight to finalized blocks; first stabilize selectable option spaces and their tag schema.
5. Prefer programmable traits over purely symbolic flavor traits when runtime behavior is intended.

## Promotion Caution
Before canon lock, compare these fragments against:
existing block, sleeve, and runtime-boundary language,
later control-room schema decisions,
and any builder implementation constraints.
