# BATCH-136 — Canon Candidates

## Status Note
These are candidate canonical fragments extracted from a raw block artifact.
Inclusion here does not itself promote them to canon.

## Candidate Fragments
1. A role-pure Instruction block should encode method, rule, or procedure rather than worldview or output style.
2. Outer block metadata should be separated into semantic, compiler, and renderer/persistence layers.
3. Example data may illustrate application without redefining the block itself.
4. Compact rule payloads such as `cantocore` may be viable for bounded instruction artifacts.
5. Raw persisted block artifacts should be reviewed as evidence for implementable canon, not ignored in favor of only abstract summaries.

## Promotion Caution
Before canon lock, compare these fragments against:
earlier glossary and compiler batches,
render-separation doctrine,
current implementation needs,
and later control-room decisions about persisted block format.
