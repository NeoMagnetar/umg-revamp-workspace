# BATCH-018 — Compiler Extract

## Current Compiler Direction Signal
This batch pushes the compiler away from hidden automatic priority winner-picking and toward surfaced structural resolution.

## Strongest Compiler Implications
### 1. Explicit pre-pass
Pre-pass should identify:
- request type
- fired gates
- directive family
- candidate sleeves/stacks
- required components
- obvious incompatibilities
- eligible routes

### 2. Compiler pass proper
Compiler pass should handle:
- deeper validation
- bundle compatibility
- conflict detection
- final runtime assembly
- trace generation

### 3. Priority redesign
The batch rejects a rigid universal priority order as the main doctrine.
Working rule:
- no competition, no priority

### 4. Adjudication direction
Proposed replacement path:
- detect conflict
- adjudicate
- suppress losers for current compile
- recompile

### 5. Staged rollout suggestion
- start with primary conflicts
- extend later to directive conflicts
- extend later to blueprint conflicts

## Risks
- governance doctrine is still unfinished
- adjudication policy is directionally strong but not final canon
- deterministic fallback ordering still needs clear trace/display rules
