1. activate sleeve
2. trace provenance
3. merge capabilities
4. add overlay
5. observe enactment failure
6. diagnose wrong ownership layer
7. patch with dedicated surface stack
8. rerun acceptance test
9. version bump the artifact

- support explicit surface-stack type distinct from reasoning-stack type
- support runtime intensity parameters for surface stacks
- support patch/version metadata for sleeve revisions
- support acceptance-test metadata attached to sleeve artifacts
- support rename/clarity cleanup when stack labels misdescribe real function

### Candidate Compiler Tasks
