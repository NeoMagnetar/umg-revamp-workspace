# BATCH-162 — Project Impact Map

## Primary Impact Zones
- **Core semantics:** sleeve identity without identity collapse
- **Runtime/compiler:** shared core plus sleeve-specific entrypoints
- **Documentation:** explain shared mind vs multiple surfaces
- **Staging/execution:** bounded local refactor workflow

## Roadmap Mapping
### Phase 1 — Stabilize core semantics
- clarifies sleeve identity as surface-specific embodiment, not separate entity
- supports the distinction between core cognition and runtime body

### Phase 2 — Map semantic decisions into compiler/runtime impact
- sleeve entrypoints should be import-safe
- runtime surfaces need explicit contracts
- shared logic should be factored into reusable modules rather than duplicated per sleeve

### Phase 3 — Documentation alignment
- add “three-sleeve Poe” explanation
- document web handler vs terminal runtime distinction
- teach sleeve separation with identity-preserving language

### Phase 5 — PRD and staging
- bounded local tarball handoff is a valid migration pattern
- staged refactor of monolith into shared-core + sleeves is a practical implementation path
