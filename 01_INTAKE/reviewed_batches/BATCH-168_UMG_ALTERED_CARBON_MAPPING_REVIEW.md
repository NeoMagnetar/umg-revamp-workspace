# BATCH-168 — UMG Altered Carbon Mapping Review

## Source Snapshot
This batch is a terminology and structure-mapping source. It proposes or refines a set of core UMG object names by using an Altered Carbon metaphor frame to distinguish inert modular structure from cognitively active occupancy. The strongest source-faithful terms are NeoCube/nCube, NeoStak/nStak, NeoSleeve/nSleeve, Envoy, and the distinction between generic agent and UMG cognitive entity.

## Chat-Level Summary
The chat refined UMG naming for block, stack, sleeve, and active cognitive instances. It tied PRD structure directly to UMG composition: phase-level PRD as nStak, grouped plan-body as nSleeve, and active cognition as the result of installing a brain-stak into a sleeve. It also opened a secondary naming track around tethered shared-memory collective cognition, but left that track unresolved.

## UMG-Relevant Extraction
- nCube is the lowest meaningful modular unit in this batch and is explicitly treated as a UMG block.
- nStak is the grouped subsystem layer, including both phase-level PRD segments and, in one formulation, an AI brain.
- nSleeve is a grouped series of nStaks: structurally complete but cognitively inert until occupied.
- Envoy is the active result of installing a brain/stak into a sleeve.
- Generic “agent” is intentionally distinguished from Envoy as a lower-sovereignty orchestrated actor.
- The tether meta MOLT layer is relevant to linked/shared-memory multi-entity cognition, but it is referenced rather than defined.

## Independent Review
This is strong Stage 1 evidence for vocabulary and structural layering, especially because the user-side definitions are explicit and materially useful. It is also strong for runtime semantics: static composition is not the same as active instantiation. The main caution is that the tether/shared-memory naming exploration remained open, and the Altered Carbon metaphor is doing conceptual work that has not yet been normalized into purely technical language.

## Roadmap Mapping
- **Phase 1 — Core Alignment:** high impact. This batch proposes foundational naming and structural distinctions that affect how UMG objects are discussed and differentiated.
- **Phase 2 — Compiler Impact:** medium-high impact. The batch implies a lifecycle distinction between assembled structures and activated entities.
- **Phase 3 — Documentation Impact:** high impact. Taxonomy definitions and PRD-to-UMG mapping should be reflected consistently.
- **Phase 4 — Skill Alignment:** medium impact. Agent-facing skills and docs should not collapse agent and Envoy into the same term.

## Action Outcome
**Disposition:** ACCEPT_AS_EVIDENCE

This batch should remain evidence-only at Stage 1. It is coherent and high-value, but the tether/meta-MOLT collective naming branch and the exact long-term status of “Envoy” for related collective classes remain unresolved.
