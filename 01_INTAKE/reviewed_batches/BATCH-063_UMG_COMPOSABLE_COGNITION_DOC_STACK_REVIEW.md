This batch strengthens the documentation and positioning side of UMG. It makes UMG more legible as a block-based framework, a modular reasoning system, and a composable cognition architecture. ## UMG-Relevant Extraction

### Core semantics
- UMG is consistently framed as a modular, interpretable, human-readable system for assembling cognition rather than writing one monolithic prompt.
- The strongest stable triad is:
  - **Primary** = main directive / objective
  - **Merge** = additive support, data, or constraints
  - **Blueprint** = tone, worldview, structure, or style influence
- UMG is repeatedly positioned as more than prompt engineering:
  - modular reasoning
  - structured thought artifacts
  - composable intelligence

### MOLT layer
- MOLT is treated as a central UMG unit, but its formal expansion is inconsistent across materials:
  - Modular Logic Token
  - Modular Operating Language of Thought
  - Modular Expression Unit
- Stable properties across definitions:
  - self-contained
  - editable
  - stackable
  - mergeable
  - reusable
- Recurrent MOLT types:
  - Primary
  - Subject
  - Philosophy
  - Instruction
- Self-adaptive additions:
  - personality MOLTs
  - style MOLTs
  - behavior pattern blocks
  - decay / reinforce / mutate dynamics

### Runtime and orchestration
- The source repeatedly describes an engine/orchestration layer that:
  - parses blocks
  - prepares a composite prompt or plan
  - injects Merge content
  - applies Blueprint influence
  - supports regeneration by editing isolated blocks
- PoeUMG adds deeper runtime language:
  - Understand → Reflect → Refine
  - context map generation
  - MOLT-driven rendering
  - reflection injection
- Self-adaptive work adds:
  - pre-response build
  - post-response adjustment
  - mutation of remembered MOLTs rather than transcript recall

### Governance and bounded control
- Strongest control layer comes from PoeUMG protocol framing:
  - instructional hard constraints
  - reflective checkpoints
  - drift detection
  - controlled mutation via vault merges
  - value-matching philosophy blocks
  - agent tethering
- The broader UMG materials imply governance through explicit modular constraints, safety blocks, compliance blocks, and traceability of which block influenced the result.

### Documentation / staging signal
- This batch strongly supports GitHub/public packaging:
  - README refinement
  - framework card usage
  - whitepaper strategy
  - MOLT standard work
  - installer and deployment language
- It also implies the need for a single reconciled documentation stack so beginner-facing UMG, NeoUMG, PoeUMG, and self-adaptive layers stop competing with each other.

## Independent Review
This is a high-value evidence batch for Phase 1 through Phase 5 work because it does not just add new terminology — it reveals where the documentation system is drifting. The stable signals are strong: Primary / Merge / Blueprint separation, modular reasoning claims, orchestration engine language, and modular governance. The unstable signals are equally important: multiple MOLT expansions, overlapping taxonomies, and mismatched abstraction layers between beginner docs, cognitive architecture docs, GPT runtime docs, and self-adaptive agent docs. This batch should be treated as evidence for reconciliation work, not as a ready-made canon layer.
