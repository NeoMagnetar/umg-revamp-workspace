# BATCH-155_CANON_CANDIDATES

## Candidate Canon Language
These are evidence candidates only. They are not canon.

- “stable base, a switchable brain, and clean escape routes”
- “minimum viable cognition first”
- “swap brains later without touching your app code”
- “one endpoint for many brains”

## Candidate Canon Ideas
- Agent-facing systems should target a stable protocol boundary rather than bind directly to a single cognition provider.
- Local and remote cognition providers should be interchangeable behind the same runtime contract where practical.
- Deployment architecture should preserve rollback safety and reversible staging.
- Memory infrastructure should remain optional unless the use case justifies activation.

## Promotion Caution
None of these candidates should be promoted from this batch alone. They need cross-batch confirmation and alignment review.
