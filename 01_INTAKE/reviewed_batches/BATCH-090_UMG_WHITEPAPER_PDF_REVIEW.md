## UMG-Relevant Extraction
### Core Semantics
- UMG is framed as modular, composable cognition rather than monolithic prompting.
- Prompt engineering is described as a construction process rather than a one-shot request.
- UMG is associated with explicit reasoning layers, portable thought objects, and human-AI co-authoring.
- UMG is described visually as an editable stack or assembly of parts.

### Roles / Taxonomy
- Primary = interprets or defines user intent.
- Merge = incorporates data, rules, context, constraints, or supporting details.
- Blueprint = applies reasoning style, tone, philosophy, structure, or strategic lens.
- The original Primary / Merge / Blueprint trinity is explicitly reaffirmed as still foundational.
- Later additions such as Meta Layer, Subject MOLT, and Instruction Node are treated as added depth rather than replacements.

### Runtime / Agent Flow
- Conceptual runtime is described as:
  - user input or intent
  - structured blocks
  - parser / UMG engine
  - LLM
  - specialized agents
  - feedback / iteration
- Specialized agents named:
  - Research Agent
  - Planning Agent
  - Persona Agent
- The runtime description is conceptual and visual, not implementation-specific.

### Documentation / Public Explanation
- Multiple reusable metaphors and explanation assets are proposed:
  - Hypercharging AI Agents with UMG
  - Lego Cognition Tower
  - Modular Mind Map
  - AI Forge
  - Orchestra of Thought
  - Time-Lapse Evolution Grid
- A screenshot-ready Cursor comparison is developed as builder-facing documentation language.
- Public explainability requirements are implied, especially around visible and correctly labeled Primary / Merge / Blueprint blocks.

## Independent Review
This batch is strongest when treated as **public-facing explanation evidence**. It does three useful things:
1. stabilizes the semantic core for communication
2. builds visual metaphors for teaching UMG
3. connects UMG to familiar AI workflow practices without replacing its own language

Its main limitation is that the runtime and parser/agent flow remain illustrative rather than formal. This means the batch should feed docs, decks, onboarding, and visual standards, not compiler canon directly.
