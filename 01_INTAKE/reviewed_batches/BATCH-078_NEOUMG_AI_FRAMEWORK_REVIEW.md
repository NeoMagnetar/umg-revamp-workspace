## UMG-Relevant Extraction

### Core semantics
- UMG is framed here as a modular cognitive architecture or behavioral operating system, not merely a prompt format.
- MOLT is defined as the Modular Operating Language of Thought and functions as a typed, self-contained logic unit.
- Composability, traceability, runtime reconfiguration, and conflict visibility are treated as central system properties.

### Runtime / execution
- Priority is mapped to runtime influence and, in the WINA integration concept, to activation budgets and neuron selection behavior.
- Merge is treated as a composition operation that should preserve or surface incompatibility rather than hiding it.
- Reflection is treated as a feedback and evaluation layer rather than just prose commentary.

### Builder / schema / tooling
- The source includes candidate block schema fields such as `type`, `content`, `priority`, `cantocore`, `id`, and `created_at`.
- Visual builder upgrades include typed blocks, operation-aware connections, priority controls, and export metadata.
- A CLI path is proposed with commands such as `umg snap`, `umg run`, `umg inspect`, and `umg edit`.

### Productization / release
- Phase 1 focuses on a simple core engine, starter block library, one legible demo, and minimal but strong documentation.
- Demo concepts include Trust Auditor, Personality Remix, Conflict Resolver, Multi-Agent Orchestra, and Time Machine.
- Adoption criteria emphasize short setup, fast first success, and visible behavior traceability.
