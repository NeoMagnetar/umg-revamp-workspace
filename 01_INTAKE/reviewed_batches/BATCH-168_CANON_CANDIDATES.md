# BATCH-168 — Canon Candidates

## Candidate Canon Language
- NeoCube / nCube = UMG block
- NeoStak / nStak = stack of related NeoCubes / subsystem / phase-level PRD unit
- NeoSleeve / nSleeve = grouped series of nStaks / complete but lifeless plan-body
- Envoy = nSleeve + installed AI brain/stak
- agent = generic non-sovereign orchestrated entity; not equivalent to UMG Envoy

## Candidate Documentation Language
- A sleeve is structurally complete but not inherently alive.
- Active cognition begins at installation/occupancy, not mere assembly.
- PRD phases can be treated as structural composition units inside UMG.

## Candidate Migration Notes
- existing generic “block / stack / sleeve / agent” language may need remapping
- public docs may need a formal taxonomy section distinguishing inert structures from active entities
- tether/shared-memory collective naming must remain open until stabilized
