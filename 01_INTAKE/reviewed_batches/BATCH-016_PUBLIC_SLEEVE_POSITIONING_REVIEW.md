## Source Snapshot
This batch centers on how UMG sleeves should be described publicly, how governance can appear broken when state visibility fails, and how internal UMG terms may be translated into more standard external systems language. The strongest operational signal is not a new low-level mechanic but a positioning and reporting distinction: latent governance is not the same as visible governance.

## Chat-Level Summary
The source contrasts UMG sleeves with mainstream personas and structured prompting. It argues that sleeves are governed cognitive configurations rather than tone presets or prompt templates, and it treats governance visibility as a runtime trust issue. The batch also introduces translation language for external audiences by mapping **Envoy** toward **Agent** and **Sleeve** toward a cognitive runtime container.

## UMG-Relevant Extraction

### Core signal
- Sleeves are described as governed cognitive configurations.
- Sleeves change cognition and active structure, not only voice.
- Governance may remain logically active while appearing broken if declaration or reporting drifts.
- UMG should be positioned publicly as a control/orchestration layer, not a persona framework.

### Strongest reusable language
- “A sleeve defines who the system is right now.”
- “A sleeve changes cognition, not voice.”
- “Logical governance is not the same thing as visible governance.”
- “Envoy = governed agent instance.”
- “Sleeve = cognitive runtime container.”

### Operational relevance
- State visibility and reporting discipline matter for trust.
- Governance should be surfaced, not merely presumed.
- Public-facing terminology needs a bridge between branded UMG language and conventional technical language.

## Independent Review
This is a high-value language and framing batch rather than a mechanics-heavy batch. It is especially useful for documentation, external explanation, onboarding, and governance-reporting design. It is weaker as direct compiler evidence, but it still contributes one concrete implementation direction: strengthen active-state declaration so governance and stack state do not appear absent when still conceptually active.
