# BATCH-173 NeoStructure Extract

## Structural Ladder in this Batch
- **nCube** = atomic publishable block
- **nStak** = composable grouped bundle / graph unit
- **nSleeve** = sovereign runtime wrapper
- **Envoy** = governance-capable agent analogue under this deployment framing

## Structural Separation
This batch strongly reinforces a two-plane system:

### Identity / Provenance Plane
- chain record
- hash / CID
- approval state
- mutation lineage
- ownership / licensing / governance metadata

### Execution / Artifact Plane
- full JSON block data
- compiled binaries
- runtime graphs
- cache and scheduler behavior
- storage and compute

## Deployment Shape
1. Builder selects or creates nCube
2. JSON/metadata pinned off-chain
3. Optional compile to runtime artifact
4. Hash/pointer recorded on-chain
5. Bundles formed into nStaks
6. Approved bundles loaded by nSleeves
7. Envoy-like runtime executes off-chain

## Structural Relevance
This batch is important because it extends UMG structure into publication, provenance, and deployment without collapsing execution into the governance layer.
