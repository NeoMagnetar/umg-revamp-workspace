- auditability

### Governance / Control
Governance was treated as a first-class architectural primitive.
The source highlights governance as supporting:
- clear conflict resolution logic
- defined authority structures
- safety and boundary encoding
- controlled autonomy

Weighted dominance fields were named as a notable control feature, though not defined in detail here.
The evaluative claim was that few creative or agentic frameworks formalize governance this centrally.

### Output Blueprint / Rendering Separation
The source distinguished internal control structure from application surface by framing UMG as a meta-layer rather than an app.
It emphasized that output quality is shaped by:
- constraints
- execution order
- field weighting
- governance structure

This implies a meaningful separation between specification/control layers and visible generated output, even though that separation was not formalized in canonical terminology inside this chat.

The source states that UMG's viability depends on:

It presents determinism as depending on:
