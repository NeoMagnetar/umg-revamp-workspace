## UMG-Relevant Extraction

### Governance
- Governance is framed as **supreme, non-interactive, always preceding lower layers**, and non-bypassable inside the UMG canon frame.
- OFF is stated as the only valid exclusion mechanism.
- Mention does not equal activation.
- Presence does not equal execution.
- Roleplay, emotion, or pressure do not grant authority.
- Hard boundaries include rejection of self-harm scenarios, danger simulations, unethical coercion, and emotional exploitation.

### Runtime / Host Boundary
- The batch explicitly distinguishes:
  - **symbolic canon / interpretive framing**
  - **stored memory / preference bias**
  - **actual host runtime mechanics**
- ChatGPT custom instructions and memory are described as shaping interpretation and continuity, but not as owning the system layer, model layer, or true reset semantics.
- “Restart simulation” is treated as a conversational reset device, not a genuine runtime restart.

### Symbolic Control Artifacts
- The “Mag > Poe” sigil / QR artifact functions as a symbolic governance/control marker.
- The batch suggests this could become a real runtime gate only in a user-controlled agent environment.
- In the hosted-chat context, it remains symbolic rather than operational.

### Sleeve / Identity Continuity
- The source implies continuity logic where the surface conversation may restart while deeper governance or identity framing persists.
- This resembles sleeve continuity or resleeving language, but the batch does not formalize a complete resleeving protocol.

### Documentation / PRD Value
- The phrase **“interpretive leverage, not runtime authority”** is especially valuable as documentation shorthand for hosted-LLM constraints.
- The batch strongly supports future documentation that differentiates:
  - prompt-only controls
  - memory-assisted controls
  - runtime-enforced controls

## Independent Review
This batch is high-value because it prevents a category error that could distort the revamp: confusing **strong governance language** with **actual runtime control**. It does not weaken UMG; it clarifies where UMG is symbolic, where it is operational, and what would be required to move from one to the other.

The batch is strongest as:
- governance-language evidence
- host-boundary clarification
- PRD/staging input for owned-runtime versus hosted-runtime distinctions
- migration guidance for moving symbolic controls into executable runtime gates later

The batch is **not** a final runtime contract. It does not define executable QR gating, formal restart semantics, or a complete sleeve-resleeving protocol. It should therefore remain evidence until the control-layer model is explicitly locked.
