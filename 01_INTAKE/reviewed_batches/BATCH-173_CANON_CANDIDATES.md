# BATCH-173 Canon Candidates

## Candidate Language
- “Put only fingerprints on-chain, keep the heavy code, storage, and compute off-chain.”
- “UMG blocks can be minted, bought, licensed, remixed, forked, and evolved while preserving provenance.”
- “Blockchain acts as identity, provenance, and governance layer; execution remains off-chain.”
- “UMG blocks are modular building blocks of thought that can be published, recombined, and evolved.”
- “Execution engine should remain pluggable.”

## Candidate Documentation Directions
- Plain-language explanation of provenance vs execution
- Builder publish-flow guide
- Definitions for nCube, nStak, nSleeve, Envoy in deployment context
- Chain preset and runtime-engine preset language

## Hold Conditions
Do not promote these directly to canon until the project resolves:
- whether provenance is core UMG or optional deployment layer
- whether marketplace/governance is near-term or long-term
- whether MAX/Mojo is preferred driver or only one supported path
