# BATCH-162 — Canon Candidates

## Candidate Language
- “Same mind. Different mouths.”
- “Each one represents a sleeve, not Poe himself.”
- “Poe is a constellation of agents, not a single file.”

## Candidate Documentation Concepts
- one shared core with multiple runtime sleeves
- UI/browser rendering is not the same thing as terminal prompting
- preserve identity across multiple surfaces without collapsing all runtimes into one file

## Candidate Runtime Rules
- CLI files should not auto-run when imported
- web sleeves should expose handler-oriented contracts
- shared cognition should be extracted into reusable lib modules

## Caution
These are strong candidates, but still evidence only. Naming, module boundaries, and post-refactor validation remain unsettled.
