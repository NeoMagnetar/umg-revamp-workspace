# BATCH-077 — Blocks / MOLT Extract

## Explicit Role Taxonomy in This Batch
- Primary - Main Block
- Merge - Support Info
- Blueprint - Inspiration

## Practical Role Meanings
### Primary
- main intent
- target output ask
- dominant content directive

### Merge
- additive support information
- material that should be woven into the output
- not a replacement for the primary directive

### Blueprint
- style or inspiration layer
- render-shaping influence
- not necessarily literal inserted content

## Example Mapping from Source
- Business Plan -> Primary
- Authentication System -> Merge
- Art of War -> Blueprint

## Candidate Block Schema
- id
- title
- role
- content
- style
- active

## Additional Block-Family Signal
- Skeleton Blocks appear separate from pure generation blocks.
- This implies the app may support more than one block family:
  - content-generation blocks
  - code/scaffold blocks
