# BATCH-099_BLOCKS_MOLT_EXTRACT

## Scope
MOLT / metadata / block-schema signals extracted from BATCH-099.

## Expanded Role Set Signal
The batch reports one of the clearest expanded UMG role lists so far:
- Primary
- Subject
- Instruction
- Directive
- Philosophy
- Blueprint
- Trigger
- Merge
- Off

This is important because it extends earlier four-role emphasis into a richer block ecology with runtime-facing functions.

## Functional Definitions Reported In-Source
- Primary = main goal / core intent
- Subject = context / domain
- Instruction = constraints / instructions
- Directive = strategic behavior guidance
- Philosophy = values / ethics / tone lens
- Blueprint = output structure / format / style
- Trigger = conditional activation
- Merge = knowledge / context injection
- Off = disabled logic

## Structural Tensions
- the expanded role set may not yet be reconciled with earlier narrower MOLT formulations
- Philosophy and Blueprint are treated as distinct layers, but precedence rules are still unsettled
- some named blocks are philosophy-heavy and may function more as canon-candidate language than executable schema units
- `molt_type` is treated as required, but the exact allowed enum set may still be evolving across batches

## High-Value Documentation Targets
- reconcile core versus expanded role taxonomy
- document functional differences between Directive and Instruction
- document Blueprint as a rendering/output layer role rather than generic formatting advice
- define Merge and Off as first-class runtime-sensitive block types
- stabilize schema fields against earlier metadata-policy work
