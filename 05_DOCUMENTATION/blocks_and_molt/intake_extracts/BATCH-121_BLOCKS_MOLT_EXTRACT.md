# BATCH-121_BLOCKS_MOLT_EXTRACT

## Blocks / MOLT Signal
- The atomic unit is the Block.
- A block is semantic, not executable.
- A block answers exactly one cognitive question.
- A block belongs to exactly one type.
- Types are fixed and non-negotiable in this packet.
- Types cannot be invented.
- Types cannot be merged at the role level.

## Role Mapping
- Purpose / Why
- Subject / About what
- Directive / How (strategy)
- Instruction / Under what rules
- Philosophy / Under what values
- Blueprint / In what form
- Trigger / When eligible

## Structural Constraints
- No block answers two cognitive questions at once.
- A block never implies time.
- A block never infers intent.
- A block may be referenced, constrained, excluded, or merged into a new block, but not edited in place.

## Caution
- This batch contains an unresolved tension:
  - role-types are non-mergeable
  - but blocks may be merged into a new block
- The relation between role integrity and merge output is not elaborated here.
