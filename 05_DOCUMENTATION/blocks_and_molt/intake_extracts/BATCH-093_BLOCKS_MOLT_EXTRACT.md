# BATCH-093 Blocks / MOLT Extract

## Scope
Block taxonomy, role expansion, stack composition, and output/internal separation signals.

## Core MOLT Surface
- Primary
- Subject
- Instruction
- Philosophy

## Extended Role Surface Mentioned
- Trigger
- Tactic
- Persona (paused)
- X blocks
- Backpack

## Structural Claims
- Blocks are modular logic units.
- Blocks can be stacked, merged, updated, and prioritized.
- Bundles/vaults group blocks for runtime loading.
- Minimal and full block bundles are treated as different runtime weight classes.

## Internal vs External Layering
- Internal layer: CantoCore compressed logic
- External layer: prose / human-readable output
- Dual-format blocks existed in one stage, then shifted toward pure CantoCore minimal forms.

## Questions This Creates
- Which roles are canonical versus optional?
- Are bundles a first-class UMG concept or a deployment/container convenience?
- Should MOLT documentation include extended roles now or hold them pending semantic review?
- Is dual-format documentation a temporary bridge or a permanent design pattern?

## Recommended Documentation Target
Map core four roles separately from extended/provisional roles so the documentation does not accidentally canonize every surfaced block type.
