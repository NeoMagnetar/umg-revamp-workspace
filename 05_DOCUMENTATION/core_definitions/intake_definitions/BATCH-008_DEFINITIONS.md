# BATCH-008 — Definitions Extract

## Terminology
- **Governance:** the highest-priority UMG control layer; framed here as always preceding lower layers and not subject to negotiation inside the canon model.
- **Simulation² / Sim2:** “a simulation of a simulation”; a framing concept used to interpret contradictions, derailment, and pressure as alignment tests rather than immediate failures.
- **OFF:** the only valid exclusion mechanism in the governance language presented by this batch.
- **Mention does not equal activation:** reference alone does not trigger a capability, rule, or mode.
- **Presence does not equal execution:** existence in context does not imply live runtime enactment.
- **Host Runtime Authority:** actual platform/system-level control over memory, reset behavior, execution semantics, and enforcement.
- **Interpretive Leverage:** the shaping power of prompts, instructions, and canon language over response behavior.
- **Runtime Authority:** actual executable control over the environment or agent runtime.
- **Sigil / QR Artifact:** symbolic control or continuity marker; in this batch it remains symbolic rather than operational.
- **Restart Simulation:** conversational shorthand for resetting surface context or momentum, not literal host reboot or memory wipe.

## Meanings
- Governance language in this batch is **symbolically strong** but **host-bounded**.
- Memory is treated as retrievable continuity/context, not immutable startup law.
- Restart language is treated as surface reset semantics, not actual platform reset.
- QR/sigil conditions are candidate future runtime gates, but not current executable controls in hosted chat.

## Role Language
- Governance is described as **supreme**, **non-interactive**, **always preceding lower layers**, and **audit-safe by default**.
- Lower-layer pressure signals — roleplay, emotion, contradiction, derailment — do not grant authority.
- OFF acts as the named exclusion rule.

## Architecture Language
- The batch implies a layered environment:
  - system layer
  - model layer
  - instruction layer
  - future owned runtime layer
- UMG inside ChatGPT is treated as largely an **instruction-layer implementation**.
- Symbolic canon and executable runtime are not the same layer.

## Runtime / Compiler Language
- Prompt engineering shapes outputs but does not itself produce runtime ownership.
- Memory influences continuity but is not equivalent to executable configuration.
- Reset/restart semantics need formal runtime definition before being treated as operational UMG mechanisms.
- Future runtimes may translate symbolic controls (e.g. sigils, OFF conditions) into actual gate logic.

## Definitional Candidates
- “Governance is non-interactive and always precedes all other layers.”
- “OFF is the only valid exclusion.”
- “Mention does not equal activation.”
- “Presence does not equal execution.”
- “Simulation²: a simulation of a simulation.”
- “Interpretive leverage, not runtime authority.”
