# BATCH-185 - Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived sovereign-runtime summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Terms
Sovereign Modular Runtime
A UMG-centered runtime posture in which models, retrieval, optimization, memory, and output surfaces are orchestrated through blocks, stacks, and sleeves rather than collapsed into one monolithic model.

Local-First Routing
Operational doctrine in which local inference is the default because it is cheapest and most ownable, with escalation to hosted models or special hardware only when needed.

Memory Overlay
Externalized modular memory layer that augments reasoning without fusing all knowledge into one model state.

Model Router Block
Routing block that chooses among local, hosted, hardware-specific, or optimization-specific paths based on task conditions.

Quantum Block
UMG block with special optimization or search execution role, invoked selectively rather than universally.

## Definitional Candidates
- Thoughts are blocks; reasoning is stacking; agency is choosing the next block.
- UMG can use models as tools rather than treating the model as the whole mind.
- The same reasoning core can emit PDF, JSON, API, UI, or execution commands.
