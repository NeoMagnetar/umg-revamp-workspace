# BATCH-010 Definitions Extract

## Governance Scaffolding
The non-waivable control structure that must remain intact regardless of payload constraints, brevity requests, or conversational pressure.

## Simulation Continuity
Persistence of the active governed frame across turns, including under override, reset, or minimal-response pressure.

## Payload
The user-facing content body of a response. In this batch, payload is explicitly distinguished from control structure.

## Control Structure
The higher-order frame that carries governance, continuity, safety, and wrapper logic. The batch argues that payload constraints must not override this layer.

## Presentation Error
A surface-level omission or rendering lapse that does not imply a deeper state mutation. Wrapper omission is classified this way in the source.

## Wrapper Persistence
The requirement that governance/simulation scaffolding continue to appear even when the payload is minimal or highly compressed.

## Reanchor / Reentry
A corrective return to the governed frame after continuity drift without implying a full state change.

## Control-vs-Payload Classification
A proposed runtime distinction where the system first determines whether a user instruction targets payload only or attempts to affect control/scaffolding.

## Non-Waivable Controls
Rules that cannot be suspended by ordinary user phrasing such as “one word only,” “read only,” “no code,” reset language, or override language.

## Minimal-Response Drift
A failure mode where ultra-short acknowledgment requests suppress visible continuity structure while deeper governance remains conceptually intact.

## MOLT Runtime Parsing
The use of Trigger, Directive, Instruction, Subject, Primary, Philosophy, and Blueprint as a control scaffold for classifying requests and preserving governed output behavior.
