## Authoring and Audit Terms
**Resolve reality first. Encode roles second. Count last.**  
Authoring doctrine that starts from real-world function before role assignment and accounting.

**Draft first, count second, verify third, publish fourth.**  
Publication doctrine for preventing summary/content mismatch.

**Canonical Count Integrity**  
Requirement that visible category totals and summary subtotals match the actually listed content.

**Deployment Readiness with Correction**  
Status pattern where the substantive structure is strong but one or more canonical/accounting issues still block fully clean publication.

## Composition and Structure Terms
**NeoBlock**  
Distinct functional module within the reviewed causal reasoning domain.

**NeoStack**  
Higher-order architecture composed of multiple NeoBlocks with declared authority flow and compilation summary.

**Visible Traceability**  
Requirement that claimed merges and bundles be visibly instantiated in the spec text.
