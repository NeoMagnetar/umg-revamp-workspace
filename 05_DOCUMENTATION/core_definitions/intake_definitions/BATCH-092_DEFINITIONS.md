# BATCH-092 Definitions Extract

## Terminology
### Agent
- Acceptable singular term in the batch.
- Treated as standard language; the batch does not require singular renaming.

### Bot
- Acceptable singular synonym in the batch.
- No special UMG-only distinction is established here.

### Envoys
- Proposed plural term for multiple agents operating together in a shared mission / stack context.
- Carries a joined, multiple, coordinated sense rather than just a count label.
- Status in this batch: candidate terminology, not canon.

### Block
- Candidate relation: a block can be an agent on a 1:1 basis.
- Exact difference from sleeve remains unresolved in this batch.

### Sleeve
- Candidate relation: a sleeve can be an agent on a 1:1 basis.
- Treated as agent-bearing rather than merely cosmetic in this batch.
- Resleeving is mentioned only as exploratory metaphor, not settled policy.

### Stack
- Candidate relation: a stack is more than one always.
- Implies an inherently plural / multi-agent structure.
- Strong claim; requires later validation against broader UMG material.

### Stack-space / shared space
- Used to indicate a shared operational context for envoys.
- Operational meaning remains unresolved: could imply shared mission state, shared memory, shared execution context, or some combination.

### Snap stack merge
- Mentioned as a design surface under investigation.
- Not fully defined in the batch.

### Anti-snap / snap-back
- Candidate coordination behavior where one envoy redirects another back toward mission adherence after drift.
- Operational thresholds, authority, and conflict rules are unresolved.

## Role Language
### Driver
- Execution-oriented envoy role in the tandem model.

### Navigator
- Monitoring / planning / correction-oriented envoy role in the tandem model.

### Tandem envoys
- Two envoys sharing one mission while holding different task responsibilities.
