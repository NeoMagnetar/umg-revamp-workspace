# BATCH-118_DEFINITIONS

## Core Positioning Language
- **CSL:** boundary / jurisdiction specifying where cognition may exist.
- **UMG:** framework operating inside CSL that specifies cognition as governed, non-executing artifacts.
- **execution systems:** downstream consumers of cognition artifacts; not definers of the artifacts themselves.
- **missing formalization target:** preferred soft novelty framing in place of “new category.”
- **boundary proposal:** safer framing for CSL than stronger system-novelty claims.
- **language for cognition:** acceptable only if explicitly narrowed so it is not confused with programming languages, prompt languages, or workflow DSLs.

## Governance / Control Language
- **governance:** pre-participatory permission semantics operating before execution.
- **OFF semantics:** absolute exclusion concept referenced as central to governance framing.
- **governance-before-execution:** core differentiator, but not a full safety solution.

## Compiler / Runtime Language
- **deterministic resolution:** scoped to specification-layer resolution under identical inputs.
- **runtime stochasticity:** remains downstream and should not be collapsed into specification-layer determinism.
- **near-runtime mutation:** structural, governed, auditable update possibility; must not collapse into mere prompt editing.
- **mutation trace / diff semantics:** explicitly needed but not finalized in this batch.
- **fail-closed behavior:** identified as a needed compiler/runtime contract.

## Value-Surface Language
- **training applicability:** usable for training-related workflows without claiming new training algorithms.
- **token reduction:** plausible through reuse/factoring, but not measured or settled here.
- **auditability / portability / extensibility:** retained as high-confidence value surface.
- **cognitive control plane:** possible headline language, but still tensioned against softer “boundary + framework” phrasing.

## Negative Definitions
UMG is explicitly not:
- an algorithm
- a model architecture
- an agent framework
- a workflow system
- a reasoning engine
- a safety filter
- a prompt system
