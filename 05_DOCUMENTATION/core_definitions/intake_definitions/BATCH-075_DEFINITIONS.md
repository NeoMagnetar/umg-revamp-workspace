## Terminology and Meanings

### Block
Modular unit of cognition with a defined role such as identity, style, control, or utility.

### NeoBlock
Structured block object described with stable fields including:
- ID
- Type
- Role
- Function
- CanonTags
Extended later with:
- Weight
- dependencies/implied dependencies

### NeoStack
Grouped layer of cognition composed of multiple NeoBlocks.

### Sleeve
Full cognitive configuration composed of NeoStacks and associated block/runtime logic.

### Genesis Layer
Meta-layer responsible for creating or modifying blocks; also described as registry, mutation engine, and compiler-like surface.

### Trigger
External activation gate rather than a stack element; used to escalate internal weights and activate behavior.

### Directive
Strategic intent layer in the expanded MOLT field set.

### Instruction
Execution-level behavior or operational control field.

### Subject
Domain focus field.

### Primary
Core identity or objective field.

### Philosophy
Guiding principles field.

### Blueprint
Structure or output-pattern guidance layer; used for shaping rather than core logic replacement.

### Bundle
Grouped deterministic block set with predictable merge behavior.

### Merge
Deterministic combination logic governed by weight and trigger conditions.

### Weight Matrix
Priority/execution dominance model with a numeric range from immutable core to dormant.

### Deterministic Merge Types
Named merge modes including:
- CORE_LOCK
- DOMINANT_FORWARD
- CONDITIONAL_ESCALATION
- STYLE_OVERLAY
- Z_AXIS_DEPTH

### Simulation²
Governing framing context used as part of the control boundary.
