# BATCH-103_DEFINITIONS

## Scope Note
These definitions are extracted from a derived source summary. They are evidence candidates, not canon.

## Terminology Extract

### structured prompt architecture
- Framing for UMG as an explicitly organized system of modular logic blocks rather than a single linear prompt.

### composable, human-legible logic
- Core UMG property in which reasoning structure remains understandable and intentionally arranged by a designer.

### simulation loop
- Proposed execution extension where multiple prompt or block variants are iterated, scored, and compared before selecting an output path.

### calibration/test block
- Proposed block type used to verify or score outputs before release.

### scenario/branch block
- Proposed block type used to explore multiple possible paths or probabilistic reasoning branches.

### dynamic block tuning
- Proposed process where an agent edits Instructions or Directives after iteration, subject to unresolved governance constraints.

### hybrid composable simulation
- Proposed mode combining UMG modular structure with parallel execution, scoring, and result aggregation.

## Role Language

### Primary
- Core objective or main goal layer.

### Subject
- Domain-context layer.

### Instruction
- Constraint or rule layer.

### Directive
- Tactical behavior-guidance layer.

### Philosophy
- Ethics, tone, or governing-value layer.

### Blueprint
- Format/style or rendering-structure layer separated from reasoning content.

## Runtime / Compiler Language

### result aggregator
- Component or step that compares multiple parallel outputs and selects or synthesizes the best result.

### branch visibility
- Requirement to expose or log simulation internals so adaptive reasoning does not become opaque.

### calibration loop
- Repeated evaluation cycle intended to check output quality or compliance before release.

## Governance Language

### static control vs dynamic adaptability
- Core tension between predesigned legible structure and runtime self-adjustment.

### approval workflow for tuning
- Unresolved governance requirement for deciding whether agent-initiated block edits are permitted and how they are reviewed.
