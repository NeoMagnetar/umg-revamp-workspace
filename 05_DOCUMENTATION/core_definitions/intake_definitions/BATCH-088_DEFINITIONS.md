# BATCH-088 — Definitions Extract

## Terminology
### UMG Stack Diagram
Visual representation of layered UMG execution structure.

### Execution Channel
Interpreted output path at the bottom of the visual stack where compiled/merged logic flows into final execution.

### Block-Weighted Logic Path
Candidate language for showing how blocks influence output through weighted contribution and ordered merge logic.

### Priority-Based Merge Routing
Candidate language for describing weighted and ordered flow from block stack through merge surfaces into execution.

## Role Language
### Primary
Top-level directive block positioned at or near the top of the visual stack.

### Merge
Supporting-data integration node feeding the central processing or execution channel.

### Instruction
Constraint / formatting / control node influencing rendered output path.

### Blueprint
Not strongly explicit in the visual source, but implied as a shaping layer comparable to style/constraint influence.

## Architecture Language
### NeoStack
Layered stack with top-level directives, middle merge/instruction logic, and lower execution channel.

### Weighted Routing
Distribution of contribution strength based on token weight or visual priority.

## Definitional Candidates
- block-weighted logic path
- execution channel
- priority-based merge routing
