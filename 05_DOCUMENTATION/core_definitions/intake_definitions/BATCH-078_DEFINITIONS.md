# BATCH-078 Definitions Extract

## Terminology
- **UMG / NeoUMG**: modular cognitive architecture or behavioral operating system framing used in this source.
- **MOLT**: Modular Operating Language of Thought; a self-contained typed block of logic or behavior.
- **Priority**: integer influence field used here for runtime weighting and routing significance.
- **Merge**: combination operation and possible block type; should reconcile logic while surfacing incompatibility.
- **Overlay**: temporary situational modifier or mode switch.
- **Reflection**: self-check, scoring, or feedback loop attached to output/runtime.
- **Blueprint**: top-level framing / identity / scope / memory / structural layer in parts of this source; not fully stabilized as ordinary block role.
- **Stack**: ordered layering of behavior, including base stack, context stack, and override stack.
- **Starter Pack / Starter Library**: reusable grouped blocks or configurations used for onboarding and testing.
- **Cantocore**: compact encoded representation field attached to a block in examples.

## Meanings by Role Language
- **Primary**: central directive or main task.
- **Subject**: domain, topic, or knowledge orientation.
- **Philosophy**: tone, ethos, framing, values, worldview.
- **Instruction**: execution rules, constraints, or behavior requirements.
- **Merge**: additive or reconciliatory logic from multiple blocks.
- **Overlay**: temporary context mode.
- **Reflection**: evaluation/feedback layer.
- **Blueprint**: framing/identity layer, sometimes top-level rather than peer role.

## Architecture Language
- UMG is framed as moving from monolithic prompts toward modular thought construction.
- Runtime is described through stack parsing, block weighting, routing, merge conflict visibility, and reflection feedback.
- Builder architecture includes typed blocks, connections, metadata, priorities, and exports.
