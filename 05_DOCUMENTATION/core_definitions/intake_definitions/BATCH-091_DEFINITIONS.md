# BATCH-091 — Definitions Extract

## Terminology
### UMG
Modular system for building AI through swappable cognitive blocks; modular cognitive architecture emphasizing composability, transparency, and interchangeable cognition.

### MOLT
Atomic unit of thought; editable, stackable, and mergeable modular logic unit.

## Core Role Language
### Primary
Core directive, main objective, or central behavioral mode.

### Subject
Topic, domain, or focus orientation.

### Philosophy
Value lens, worldview, tone, or reasoning stance.

### Instruction
Behavioral limits, operational rules, or formatting/control constraints.

### Merge
Combination surface for logic and supporting data integrated into the final output.

## Extended Composition Language
### Blueprint
Identity, scope, or structural/output-style layer influencing presentation without direct content insertion.

### Instruction Node
Filter/format/control surface adjacent to output behavior.

### Meta Layer
Higher-order override or recursive reflective layer.

## Runtime / Compiler Language
### Priority
Execution state and ordering system with numeric levels.

### Token Weighting
Resource/influence allocation mechanism tied to priority.

### Merge Engine
System component that builds prompt stack, handles hierarchy, and composes final logic path.

### Context Map
Structured intermediate representation used before rendering response.

### Reflection Injection
Post-processing or recursive control step added after or during response composition.

## Agent / Memory Language
### Self-Adaptive Agent
Agent that generates MOLTs dynamically, updates weights post-response, and adapts behavior/style.

### MOLT-Based Memory
Memory model using modular active state rather than transcript-only history.

## Definitional Candidates
- MOLT = atomic unit of thought — editable, stackable, mergeable
- composable identity via swappable MOLT blocks
- modular constraints through Philosophy and Instruction blocks
