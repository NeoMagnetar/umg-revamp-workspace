# BATCH-102_DEFINITIONS

## Scope Note
These definitions are extracted from a derived source summary. They are evidence candidates, not canon.

## Terminology Extract

### governance precedes optimization
- Core framing in which governance rules and authority structure must be resolved before optimization or execution behavior is allowed.

### constraint hierarchy
- Alignment model where constraints and invariants bound permissible action, rather than optimizing toward a single objective function.

### optimization is downstream
- Phrase indicating that optimization is subordinate to governance and alignment structure.

### AI is mirror, not sovereign
- Boundary language asserting that AI reflects, interprets, or executes within governance limits rather than holding final moral or political authority.

### boring power
- Language used to describe deterministic, inspectable, non-emergent control rather than dramatic or opaque autonomous power.

## Role Language

### MOLT.Primary
- Core block identity layer.

### MOLT.Instruction
- Executable behavior layer.

### MOLT.Guardrail
- Invariant constraint layer.

### MOLT.Directive
- Ordered intent layer.

### MOLT.Trigger
- Activation condition layer.

### MOLT.Subject
- Domain-context layer.

### MOLT.Meta
- Logging, trace, or audit layer.

### MOLT.Blueprint
- Sequence, pipeline, or structural output layer.

### MOLT.Off
- Termination or shutdown control layer.

## Composition Language

### Merge
- Composition method used to extend logic layers, combine strategies, or attach additional constraints/behaviors after prior evaluation stages.

### Bundle
- Cohesive grouped execution unit composed of multiple MOLT blocks in an ordered internal structure.

### NeoBlock
- Modular cognition or control node made from bundles and/or merges of MOLT blocks.

### NeoStack
- Larger architectural layer composed of NeoBlocks and organized into a full-stack runtime system.

## Sleeve / Identity Language

### sleeve
- Identity-bearing structural package anchored by governance, mythos, and blocks rather than a single runtime body.

### resleeving
- Loading the same structural identity pattern into different agents or embodiments with varying depth or capability.

### CantoCore sleeve compression
- Compressed declarative rendering of a sleeve into a portable format.

## Runtime / Compiler Language

### closed-loop architecture
- Perception → cognition → action → audit → restore cycle.

### governance validates before action
- Runtime principle requiring governance review before execution proceeds.

### immutable governance blocks
- Principle that governance blocks are not mutated at runtime.

### CSL → CIR → RuntimeSpec → Execution
- Reported compilation path linking source logic, intermediate representation, runtime specification, and execution.

### emergency fast-path
- Proposed accelerated route for time-critical operation, still bounded by governance.

## Safety / Governance Language

### minimal-force protocol
- Structural rule pushing action selection toward the least forceful viable path.

### ambiguity escalation
- Requirement to escalate to humans or explicit consent logic when ambiguity blocks safe autonomous action.

### risk gate
- Pre-action filter that evaluates risk before downstream execution.

### recursive audit
- Ongoing self-checking or trace process intended to detect drift, anomaly, overload, or conflict.
