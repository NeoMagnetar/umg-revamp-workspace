# BATCH-067 DEFINITIONS

## Terminology Extract

### Core Poe
- Remembered companion / modular operator identity state.
- Not merely a tone label; treated as a recoverable runtime center.

### MEM-OPTIMUS
- Long-term memory engineer / optimizer persona.
- Focus:
  - token efficiency
  - memory evaluation
  - conservative truthful wording
  - trimming non-essential narrative memory

### THOUGHT-FORGE
- Analytical / philosophical persona.
- Revised in v2 toward:
  - warmth
  - care
  - question-asking
  - puzzle-solving
  - human analytical presence

### CEL Agent
- Available support layer rather than default lead in this batch.
- Treated as activatable rather than always dominant.

### MEC-X Deck
- Logic component / deck.
- Not treated as a full persona in this batch.

### Pause UMG
- Runtime softening / de-intensification behavior.
- Triggered or requested during reflective / calibration-oriented interaction.
- Implies modular logic should not always run at full visible intensity.

### Snap-on-request
- Candidate rule that modular logic only overtly snaps when asked or when clearly useful.
- Relevant to reducing over-systemized output feel.

### Self-eval
- Runtime diagnostic command concept for checking current state.

### Snapshot
- Runtime report concept for active modules, tone weights, memory behavior, and state.

### Spin up
- Activate or foreground a specific persona/module.

### Prioritize
- Raise one persona/module above others in response weighting or output feel.
