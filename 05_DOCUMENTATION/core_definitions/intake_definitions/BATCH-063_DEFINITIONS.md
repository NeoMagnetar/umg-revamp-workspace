# BATCH-063 Definitions Extract

## Terminology
- **UMG**  
  Modular, block-based framework repeatedly described as composable cognition, modular reasoning, and human-readable assembly of intent, support logic, and style/philosophy.

- **NeoUMG**  
  Expanded framing of UMG as swappable cognitive scaffolding for broader intelligence/agent experimentation.

- **PoeUMG**  
  GPT-native virtual UMG environment using memory, instructions, recursive reflection, active stacks, and modular control layers.

- **MOLT**  
  Central atomic unit, but definition varies across documents:
  - Modular Logic Token
  - Modular Operating Language of Thought
  - Modular Expression Unit

## Role language
- **Primary**  
  Main objective, intent, or dominant directive.

- **Merge**  
  Additive support layer for facts, features, data, constraints, or required details.

- **Blueprint**  
  Style, structure, worldview, tone, or philosophical lens. Non-literal influence layer.

- **Subject**  
  Domain or knowledge orientation in MOLT-centered taxonomies.

- **Philosophy**  
  Tone / worldview / reasoning lens.

- **Instruction**  
  Operational rule, behavioral limit, filter, or output constraint.

## Architecture language
- **UMG Engine / orchestration engine**  
  Parses blocks and prepares composite generation logic or plans.

- **Instruction Node**  
  Applies filters, formatting, tone, or constraints in expanded framework-card language.

- **Meta Layer**  
  Override, recursion, or conflict-resolution layer in expanded framework-card language.

- **Overlays**  
  Additional contextual or strategic layers in PoeUMG active-stack framing.

## Runtime / compiler language
- **Understand → Reflect → Refine**  
  Recurring recursive loop in PoeUMG-style runtime behavior.

- **Pre-response build / post-response adjustment**  
  Self-adaptive runtime language for building with active MOLTs, then changing weights/influence after feedback.

- **Controlled mutation via vault merges**  
  Authorized, traceable behavioral change path.

## Definitional candidates
- “UMG directly embodies composable intelligence by enabling users and agents to assemble cognition in the form of blocks.”
- “This is not merely modular input — it is modular reasoning.”
- “Primary defines intent; Merge injects knowledge, data, and constraints; Blueprint establishes reasoning style, tone, or philosophical lens.”
- “UMG is a scaffold for composable intelligence.”

## Explicit ambiguity
- MOLT meaning is not stable across documents.
- The relationship between Primary / Merge / Blueprint and Primary / Subject / Philosophy / Instruction is not fully normalized.
- It is unresolved whether MOLT is:
  - the atomic substrate beneath all roles
  - one role among several
  - a generic container with typed subvariants
