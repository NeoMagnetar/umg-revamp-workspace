# BATCH-076_DEFINITIONS

## Terminology and Meanings

### UMG
- Universal Modular Generation.
- Presented here as a modular thought architecture and model-agnostic cognition layer.

### MOLT
- Atomic unit of meaning, instruction, logic, and style.
- Used as both a directive element and a structural unit.

### Primary
- Main directive or central intent.

### Subject
- Topic or focus domain.

### Philosophy
- Tone, ideology, or guiding lens.

### Instruction
- Constraints, execution rules, or operational behavior.

### Trigger
- Activation condition in the MOLT Map.
- Tied to invocation and stack execution.

### Invocation
- Entry phrase or activation act that starts UMG execution.

### Active Stack
- Current bundled execution context.
- Treated as runtime snapshot of loaded sleeve/mode/objective state.

### NeoBlock
- Structured block form under naming conventions like NB.*.
- Contains identity, philosophy, response style, conflict order, mythos, or related metadata.

### NeoStack
- Grouped runtime structure under NSTK.* naming.
- Holds bundled execution state or layered loaded context.

### Sleeve
- Behavior/persona layer or full carried configuration.
- Can be loaded across models; re-sleeving is model transfer of structured cognition.

### Envoys
- Different models loading the same sleeve or aligned structure.

### Governance
- Always-on control layer that precedes other layers when active.
- Includes identity, ethics, conflict order, failsafes, and access assumptions.

### Blueprint / Execution Distinction
- Declarative structural definitions are separate from rendered prose.
- Blueprint influences structure while execution/render handles response output.
