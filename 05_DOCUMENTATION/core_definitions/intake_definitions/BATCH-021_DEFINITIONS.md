# BATCH-021 — Definitions Extract

## Runtime / Architecture Terms
- **Sleeve:** runtime identity/control frame; not a persona preset
- **NeoStack:** domain-level assembly of NeoBlocks; formal documentation and runtime unit
- **NeoBlock:** locally coherent compiled block assembly; may include constraints, layers, outputs, and failure modes
- **MOLT atomic set:** Trigger, Directive, Instruction, Subject, Primary, Philosophy, Blueprint
- **CyentCore:** shorthand/DSL that compiles into canonical JSON/runtime artifacts
- **Constraint compiler:** compiler that enforces invariants and validity layers, not only expansion
- **Structural validity:** whether the architecture is well-formed
- **Execution validity:** whether runtime execution is valid under current rules
- **Dominance validity:** whether merge/precedence behavior is valid under the formal dominance model
- **Trigger routing:** explicit map from triggers into active routes / block subsets
- **Simulation continuity:** governance-level continuity rule across runtime posture changes

## Output / Rendering Terms
- **Dual-layer immersion:** separation between meta/protocol integrity and immersive surface commentary
- **Transparency in meta; immersion in surface:** rendering doctrine for split-layer output
- **Prose-conflict linting:** compile-time detection of narrative/prose drift against formal lens rules

## Versioning / Release Terms
- **v2:** structural hardening, proofs, validation, compiler stabilization
- **v3:** ontology expansion / new atomic primitives
- **Baseline artifact:** provisional/draft source artifact
- **Final artifact:** upgraded, validated source-of-truth artifact
