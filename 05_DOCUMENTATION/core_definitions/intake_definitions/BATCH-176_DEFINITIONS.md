## Core Canon Terms
**Cognitive Specification Layer (CSL)**  
A layer that specifies cognition in a human-legible, machine-enforceable form without itself performing execution.

**RuntimeSpec**  
Execution-neutral compiled cognition artifact describing what downstream runtimes should receive.

**Trace**  
Mandatory chronological and audit-oriented record of compilation events and decisions.

**PrimaryShell**  
Single output anchor or output container for an active snap; a role, not a MOLT type.

## Structural Terms
**Domain**  
Current strongest candidate human-facing term for cross-type grouped cognition collections, explicitly separated from mechanical compiler stack behavior.

**Stacking**  
Compiler-level same-type competition resolution, distinct from broader human-facing grouped constructs.

**Structural Merge**  
Compiler-safe same-type combination using explicit merge keys and merge policies.

**Creative Synthesis**  
Broader combining or melding of concepts that may belong to generation/runtime behavior rather than compiler merge law.
