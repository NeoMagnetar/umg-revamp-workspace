# BATCH-007 — Definitions Extract
## Terminology
### Sleeve
- Defines what exists.
- Static identity/configuration layer rather than current-turn interpretation.
- Proposed new base form here is neutral, governed, and idle by default.

### MOLT Map
- Defines what is happening now.
- Dynamic turn-interpretation layer containing Trigger, Directive, Instruction, Subject, Primary, Philosophy, and Blueprint.
- Non-canonical and not itself a grant of capability.

### Tag
- Recall-only and non-authoritative.
- Useful for lookup and retrieval, not for independent permission or authority.

### File
- Reference-only unless explicitly promoted.
- Evidence/tooling surface, not automatically live instruction.

### Memory
- Durable process-law store.
- Should hold stable rules and law-like boundaries, not runtime state.

### Neutral Base Sleeve
- Proposed governed idle base state for UMG.
- Should host explicit modes/overlays without automatically entering design, inspection, execution, mutation, or resleeving.

## Role Language
- **Trigger** — current-turn event initiator inside the MOLT map
- **Directive** — behavior/mode steering within the current turn
- **Instruction** — local operational rule
- **Subject** — contextual/domain lens
- **Primary** — anchor/core for the current interpretation
- **Philosophy** — additive worldview or method lens
- **Blueprint** — output style/emission layer only

## Architecture Language
- authority flows downward
- governed and idle
- default-OFF invariant
- explicit activation only
- no silent mutation
- no inferred authority
- reference-only files
- recall-only tags
- static identity vs dynamic context

## Runtime / Compiler Language
- missing anchor artifacts
- system check protocol
- MOLT map schema
- tag semantics file
- validation without permission
- inspection without execution
- structural validation failure
- NL-first stabilization before syntax reduction
