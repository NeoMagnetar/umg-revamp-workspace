# BATCH-173 Definitions

## nCube / NeoCube
Atomic UMG block. In this chat it is treated as a uniquely identified modular asset that can store metadata, provenance pointers, and references to compiled artifacts.

## nStak / NeoStak
Composable grouping or bundle of related cubes/kernels. In this chat it functions as a higher-order deployable or governable unit.

## nSleeve / NeoSleeve
Sovereign runtime wrapper around grouped logic. In this chat it acts like an agent-body or smart-account-style execution wrapper with roles, permissions, and bundle import rights.

## Envoy
Sovereign agent analogue in the proposed deployment framing. It may be owner-controlled, DAO-controlled, or multisig-guarded.

## Mutation Lineage
Version history and immutable change tracking represented through hashes, event logs, or version records rather than storing all content on-chain.

## Publish Trigger
Builder action that pins JSON/metadata off-chain, optionally compiles the block, and writes a pointer/hash/provenance record on-chain.

## On-Chain Identity Layer
Lightweight layer for identifiers, hashes, provenance, approvals, and governance.

## Off-Chain Execution Layer
The heavy layer for JSON bodies, binaries, compiled artifacts, storage, GPU/CPU compute, and runtime scheduling.

## Bundle
Deployable grouped artifact made from multiple cubes/kernels, often represented as a graph or directory of related runtime assets.
