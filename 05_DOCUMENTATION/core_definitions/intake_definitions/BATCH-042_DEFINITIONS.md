# BATCH-042 — Definitions Extract

## Terminology Present in Source

### Sleeve
A profileable identity artifact / modular identity configuration that can be activated, loaded, and held until explicit switch.

### Sleeve Taxonomy
Three-family catalog structure used in this batch:
- personality sleeves
- professional sleeves
- abstract / bizarre sleeves

### Cognitive Posture Selection
Framing used to distinguish professional sleeves from merely cosmetic persona naming.

### Mode-Based Structural Switching
The idea that changing mode is a structural runtime event, not just a tone change.

### NeoStack
The immediate structural layer assigned to each sleeve in this batch. Every sleeve receives four NeoStacks.

### Activation Protocol
Operational doctrine described in the source:
- activate sleeve
- enumerate NeoStacks
- generate downstream structure
- optionally add bundles
- remain in-sleeve until explicit switch

### Persistent In-Sleeve State
Expected behavior that the system stays inside the selected sleeve until a switch command occurs.

### Rendered PDF Artifact
Human-facing packaged output requested for the sleeve catalog. Separate from the underlying structured sleeve data.

## Definitional Candidates
- modular identity separation
- cognitive posture selection
- mode-based structural switching
- persistent in-sleeve state
