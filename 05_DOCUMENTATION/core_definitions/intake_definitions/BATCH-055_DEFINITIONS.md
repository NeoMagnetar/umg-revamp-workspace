# BATCH-055 — Definitions Extract

## Terminology
### UMG
- Framed in this batch as Mag's architecture for modular AGI-oriented systems.
- Associated with memory, recursion, contextual adaptation, and evolving agent behavior.

### Mag
- Described as inventor of UMG and origin point for the system's design direction.

### Poe
- Described as a GPT agent operating across projects through stored materials such as vaults, journals, and instructions.

### JSON vaults
- Storage surface for logic expressed as MOLT blocks.

### MOLT
- Logic-block format referenced in JSON-vault storage.
- Explicitly named roles in this batch:
  - Primary
  - Subject
  - Instruction
  - Philosophy

## Role Language
### Primary
- Purpose or identity-bearing logic.

### Subject
- Domain or contextual focus.

### Instruction
- Action-guiding logic.

### Philosophy
- Value or worldview logic.

## Architecture Language
### Long-term instructions
- Higher-scope instruction surface with broader operational reach across sessions or contexts.

### Project instructions / project context
- Lower-scope or more locally bounded instruction surface associated with a specific project context.

### Memory
- Mentioned as a separate concept from instructions and project context; exact boundary not finalized in this batch.

### Scope hierarchy
- Emerging concept that storage location affects runtime scope, persistence, and behavioral influence.

## Runtime / Compiler Language
### Trigger
- Operational activation path; example restated as storing/syncing immediately when the user says “save block.”

### Priority
- Recency-weighting rule for block frequency and recent use.

### Merge
- Conservative merge posture:
  - snap/merge when logic matches
  - merge new logic only if superior
  - resolve clashes explicitly rather than auto-overwrite

### CantoCore
- Internal dialect or compressed language for token-efficient thought stitching.
- Used here as a compression layer for system identity/instruction material.

### Render mode
- Distinction between compressed/internal syntax and normal user-facing prose voice.

## Definitional Candidates
- UMG as architecture/scaffolding for memory, recursion, and contextual adaptation.
- CantoCore as an internal representation layer rather than default public output.
- Long-term instructions and project context as separate runtime scopes, not interchangeable containers.
