# BATCH-064 Definitions Extract

## Terminology
- **UMG**  
  Modular system for assembling complex outputs and reasoning structures through role-separated blocks.

- **NeoUMG / NeoModGen**  
  Broader composable-intelligence / swappable-cognition framing, including open-source and architecture-level categories.

- **PoeUMG**  
  GPT-native modular cognition environment using active stacks, MOLT blocks, recursive reflection, and alignment controls.

- **MOLT**  
  Atomic unit of thought in the surfaced framework-card material, editable, stackable, and mergeable.

## Role language
- **Primary**  
  Main directive, objective, or dominant operational mode.

- **Merge**  
  Additive support layer for facts, features, data, constraints, or required details.

- **Blueprint**  
  Style, philosophy, tone, structure, or worldview layer that influences output without usually being inserted verbatim.

- **Subject**  
  Theme, topic, or knowledge orientation in MOLT-centered runtime taxonomies.

- **Philosophy**  
  Tone, worldview, logic directive, or reasoning lens.

- **Instruction**  
  Operational rule, behavioral limit, filter, or execution constraint.

## Architecture language
- **Instruction Node**  
  Filter + format layer in framework-card terminology.

- **Meta Layer**  
  Override / recursive logic / conflict-resolution layer.

- **Overlays**  
  Additional contextual or strategic layers shown in PoeUMG active-stack examples, but not fully formalized in the surfaced sources.

- **Active Stack**  
  PoeUMG-style runtime identity/configuration composed of swappable MOLT blocks and overlays.

## Runtime / compiler language
- **UMG Engine**  
  Combination/orchestration engine that prepares composite generation logic or plans.

- **Input Parsed → Context Map Generated → MOLT-Driven Response Rendered → Reflection Injected**  
  PoeUMG execution sequence surfaced in the source pack.

- **Pre-Response / Post-Response Loop**  
  Self-adaptive runtime behavior where replies are built from active MOLTs and then reweighted after feedback.

## Definitional candidates
- “UMG is not merely modular input — it is modular reasoning.”
- “Each block is a cognitive building unit.”
- “UMG is a scaffold for composable intelligence.”
- “MOLT: Modular Expression Unit — the atomic unit of thought.”

## Explicit ambiguity
- Cantocore is only partially evidenced in the surfaced materials.
- Overlay semantics and ordering rules are not fully formalized.
- The relationship between user-facing block roles, runtime MOLT roles, and broader architecture shells is not normalized.
- No final memory-ready schema or canonical instruction format was selected in this chat.
