# BATCH-135 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived block-architecture summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Artifact Terms
**NeoBlock**  
A named, scoped cognitive artifact with explicit role and output contract.

**NeoStack**  
A multi-domain work architecture for a role or workflow.

**NeoSleeve**  
A mountable higher-order skill container or specialization layer that can be inserted into a more agnostic sleeve.

**Schema Block**  
A block whose purpose is to define structured data and validation boundaries.

**Process Block**  
A block that defines an exact order of operations, phase gates, and required artifacts.

## Architectural Doctrine Terms
**Fractal Decomposition**  
Principle that sleeves, stacks, blocks, and important sub-artifacts may recursively expand into schemas, validators, interfaces, traces, and file trees without breaking architecture.

**Block-Native Sections**  
Section names and internal modules that arise from the block’s real responsibility rather than from a generic template.

**Skillpack Mounting**  
Pattern in which an agnostic base sleeve hosts a specialized NeoSleeve or NeoStack cartridge.

## Rendering and Compiler Terms
**Semantic Render Structure**  
Controllable text-level ordering, labels, field rhythm, line breaks, and section cadence.

**Renderer/UI Control**  
Typography-level control such as font size, weight, kerning, and pixel spacing that belongs to a dedicated renderer layer.

**Adapters Translate Shape; Blocks Enforce Meaning**  
Principle separating transport/format conversion from semantic law enforcement.

## Definitional Candidates
A serious block deserves its own internal architecture.
