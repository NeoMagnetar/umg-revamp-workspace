# BATCH-104_DEFINITIONS

## Scope Note
These definitions are extracted from a derived source summary. They are evidence candidates, not canon.

## Terminology Extract

### world model
- Predictive simulation subsystem that represents environments, outcomes, or likely future states, but does not itself decide or authorize action.

### cognitive operating system
- Framing for UMG as the broader architecture that governs state, memory, planning, translation, and execution rather than only generating predictions.

### prediction does not equal decision
- Core boundary claim that predictive output must remain subordinate to governance, planning, and permission surfaces.

### governance precedes intelligence
- Ordering principle that constraints, permissions, and authority checks come before simulation or reasoning influence action.

### translation to operations
- Boundary layer that converts plans into commands, tool calls, API actions, or language output.

## Role Language

### Context & State (MOLT)
- State-bearing layer holding current situation, historical continuity, active constraints, and agent state in explicit legible form.

### WorldModelBlock
- Candidate block family representing environment simulation or predictive modeling capability inside UMG.

### PredictiveBlock / StateMap
- Candidate naming for predictive or state-bearing structures; boundary between these and WorldModelBlocks remains unresolved in the source.

### Planning & Action Blocks
- Candidate block family responsible for turning constrained state plus simulation outputs into actual plans or executable actions.

### Governance Blocks
- Candidate block family defining authority checks, permissions, prohibitions, and constraint logic.

## Runtime / Compiler Language

### layered pipeline
- Ordered flow from intent through governance, context, simulation, planning, translation, and execution.

### compilation boundary
- Implied transition where structured intent and planning become executable operational output.

### execution feedback loop
- Implied loop where action results become future input/state, though no formal update mechanism is defined here.

## Governance Language

### explicit governance
- Constraint and authority logic that is directly represented rather than assumed to emerge from prediction.

### simulation cannot bypass constraints
- Safety condition requiring predictive layers to feed planning rather than trigger execution directly.
