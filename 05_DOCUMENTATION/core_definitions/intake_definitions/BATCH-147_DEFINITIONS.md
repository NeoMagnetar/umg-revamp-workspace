# BATCH-147 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived generation-protocol summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Workflow Terms
**UMG Block Generation Protocol**  
Standardized process that receives category input, converts items into normalized UMG JSON blocks, writes files to structured directories, zips outputs, and reports progress through a status overlay.

**Path-Aware Category**  
Category field whose value is treated not just as a label, but as a logic-domain path supporting routing, organization, and downstream export structure.

**Subcategory Zip**  
Per-subdomain packaged archive containing the generated block files for one specific category slice.

**Citadel Block Inventory**  
Large file-backed library of generated UMG block units intended for later runtime, sidebar, canvas, or library use.

## Schema Terms
**block_id**  
Placeholder-oriented block identifier that may later be canonicalized or hashed.

**ledger**  
Structured provenance object containing authorship, verification, timestamp, fork lineage, and edit history.

**export_config**  
Export metadata for documentation or rendering targets distinct from the core JSON block structure.

## Definitional Candidates
These blocks are living modular intelligence units.  
A block can be inspectable, draggable, editable, mergeable, and exportable.  
Future blocks may include trigger, function_calls, or AI instructions.
