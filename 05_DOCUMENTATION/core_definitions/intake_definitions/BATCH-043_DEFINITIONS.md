# BATCH-043 Definitions Extract

## Instruction Layer
A persistent control substrate that governs how prompts, stacks, and frameworks interact, persist, and constrain each other over time.

## Prompt
An output-generating layer; not the full control architecture.

## Stack
A reasoning-mode organizer; not by itself the persistence/control substrate.

## Governance
The stable authority/control anchor that remains active unless genuinely mutated or degraded.

## Presentation Layer
The rendering/output-shape layer where envelope compliance and formatting occur.

## PRESENTATION_ERROR_ONLY
A structural failure class indicating that output/rendering format failed while governance, sleeve identity, authority, and safety remained intact.

## Simulation Continuity Rule
A recovery rule where envelope/rendering failures are corrected on the next turn without implying state mutation or governance drift.

## Structured Governance + Activation Rules
A framing for UMG memory/control in which persistence is treated as architecture and activation behavior rather than as stored text alone.

## Meta-Cognitive Policy
The policy layer that governs how frameworks compose, persist, and constrain one another over time.
