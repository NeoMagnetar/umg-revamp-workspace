# BATCH-005 — Definitions Extract
## Terminology
### Block
- Atomic unit of meaning with role, content, tags, and priority.
- Semantic building block rather than a full runtime mode.

### Stack
- Ordered set of blocks for a domain or behavior.
- Priority/order logic exists both inside stacks and across active stacks.

### NeoBlock
- Compiled active view of ordered and selected blocks inside a stack.
- Runtime-facing object carrying ordered block ids, active selections, and bundle participation.

### NeoStack
- Compiled aggregation of NeoBlocks by active domain/runtime grouping.
- Operational grouping rather than passive labeling.

### Sleeve
- Top-level cognitive configuration containing governance, active stacks, and defaults.
- In this batch, can also correspond to a project-level runtime identity.

### Governance
- Immutable or non-overridable control layer above stacks and style.
- Distinct from cognition and formatting.

### Blueprint / BP_MODE
- Output-emission layer.
- Selects how a response is rendered rather than how cognition is organized.
