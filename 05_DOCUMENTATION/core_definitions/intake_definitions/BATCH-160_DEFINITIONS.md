# BATCH-160 — Definitions Extract

## Definitions surfaced in source
- **UMG** = modular cognition built from self-contained logic blocks and runtime layers.
- **MOLT** = Modular Operating Language of Thought.
- **Snap** = combine blocks of different types into an executable agent/workflow.
- **Merge** = combine same-type logic or supporting content into stronger or more complete form.
- **Stack** = arrange blocks into prioritized memory/execution layers.
- **Alignment Core** = support block enforcing ethical alignment.
- **Blueprint** = style / reasoning / structure guidance.
- **CEL** = Creative Extension Layer; ideation / mutation / creative extension.
- **Chaos Lens** = disruptive creativity / recombination layer.
- **Overlay** = runtime modifier that changes tone / style / logic without changing core alignment.
- **Meta block** = narrative / self-reflection / memory / engine configuration layer.
- **Alignment Spine** = structured alignment layer injected at runtime and versioned for deployment.
- **Sleeve** = structured persona/runtime identity layer with overlays, style, and caps.

## Structured artifact definitions preserved
- **Block spec** = canonized JSON object with fields such as `id`, `version`, `inputs`, `outputs`, `contract`, `triggers`, and `tests`.
- **Sleeve spec** = structured identity/runtime object with fields such as `id`, `overlays`, `style`, and `caps`.
- **Training row** = JSONL sample structure that can reference `blocks_used`, `sleeve`, `spine`, and `tags`.
- **Canonical atlas** = the proposed single source of truth that unifies canon before training.
