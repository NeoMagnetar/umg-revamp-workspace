- **Primary** — main directive / main block / core output ask

- **Blueprint** — inspiration / style lens shaping render, not necessarily copied literally

## Architecture Language
- **Block Manager / Command Center** — front-end control surface for modular generation
- **Skeleton Blocks** — block family apparently oriented toward code/file scaffolds rather than pure text prompts
- **active block** — block selected/enabled for current generation context

- **merge controls** — richer merge behaviors advertised as `blend`, `dominate`, `reference`
- **immediate generation loop** — edit blocks/instructions -> regenerate output
- **transformation stage** — possible stage where style/format modifiers such as Binary are applied
