# BATCH-036 — Definitions Extract

## Terminology and Meanings

### Cognitive control architecture
A framing term for UMG as a layered system that governs, routes, structures, and constrains cognition rather than merely styling output.

### Cognitive operating system
A high-level analogy for UMG as a system containing governance, runtime environment, process-like stacks, lower-level blocks, and output execution boundaries.

### Cognitive specification layer
A candidate term for the project’s declarative architecture layer: the layer that specifies cognition structure, governance boundaries, execution ordering intent, and separation from emitted execution.

### Control shell
A phrase used to describe UMG as the structured layer wrapped around the inference engine.

### Inference engine
The underlying LLM/GPT substrate that performs generative reasoning but does not, by itself, guarantee structured control, state, validation, or governance.

### Prompt grammar
A framing that emphasizes architecture, ordering, and structured control declarations rather than loose natural-language prompt narrative.

## Role / Architecture Language

### Governance
Top-level control and constraint layer that should remain external to mutable runtime behavior and should eventually be enforceable rather than merely suggested.

### Sleeve
Runtime environment / session-configuration layer that shapes active operating mode and contextual identity.

### NeoStack
Functional reasoning cluster or process-like subsystem inside the broader cognitive architecture.

### NeoBlock
Mid-level functional unit inside a NeoStack.

### MOLT block
Lower-level atomic control / reasoning unit inside the broader structure.

### Execution output
Rendered downstream response or action layer, explicitly separated from cognition specification.

## Runtime / Compiler Language

### Declarative architecture
A system whose structure is described in prompt/spec form but not yet enforced by an external execution engine.

### Executable runtime/controller
A future system component responsible for ordered stage execution, validation, state persistence, and governance enforcement.

### State persistence
The capacity for architecture state, working context, and controlled continuity to exist outside one probabilistic generation step.

### Validation
An explicit runtime/controller stage that checks structural, policy, or execution correctness before advancing or emitting output.

## Definitional Candidates
- UMG = control architecture; LLM/GPT = inference engine
- UMG inserts a control layer between prompt and cognition
- cognition as a first-class object, viewable, transparent, and separate from execution
- prompt grammar rather than prompt narrative
