# BATCH-012 — Definitions Extract

## Terminology
- **Block** — atomic unit of meaning; identified by id, role, content, and tags; describes cognition rather than execution.
- **NeoBlock** — compiled active view of blocks within a stack.
- **NeoStack** — domain-level aggregation of NeoBlocks; a mode of cognition or work.
- **Sleeve** — the active system identity containing governance, active stacks, defaults, and constraints.
- **Bundle** — grouping structure for ordering or alternation.
- **Merge** — combines compatible blocks while preserving identity.

## MOLT Role Language
- **TR / Trigger** — activation or event-entry condition.
- **DIR / Directive** — strategic behavior or route.
- **INS / Instruction** — operational rule layer.
- **SUB / Subject** — domain or contextual lens.
- **PRI / Primary** — dominant anchor function.
- **PHI / Philosophy** — worldview or stance layer.
- **BP / Blueprint** — output/emission shaping.

## Architecture Language
- Request handling begins with a short MOLT skeleton.
- Sleeve modes activate explicitly.
- Activated modes enumerate NeoStacks.
- Each NeoStack yields one Primary NeoBlock plus layered Instructions and optional Bundles.
- Blueprint selection is separate from cognition selection.

## Runtime / Compiler Language
- Runtime is staged rather than monolithic.
- Six-phase direction present in the batch:
  - Initialize
  - Goal Genesis
  - Goal Selection
  - Optional Plan/Execute
  - Output Emission
  - Runtime Self-Improvement
- Typed contracts present for CTX, GoalCandidate, GoalGenesisOutput, GoalSelectionOutput, and SelfImprovementOutput.
- Governance lock is runtime-relevant, not only documentary.
- Drift thresholds, anti-thrash locks, and audit traces are explicit runtime concepts.

## Role / Control Language
- Governance anchor remains central.
- Exactly one format blueprint should be active per response.
- MODE and BP_MODE are orthogonal.
- Safety policy is embedded into runtime selection and improvement logic, not left as an afterthought.

## Definitional Candidates
- “Format is controlled independently from cognition.”
- “Exactly one format blueprint is active per response.”
- “MODE selects how the system thinks; BP_MODE selects how output is emitted.”
- “Describes cognition, not execution.”
