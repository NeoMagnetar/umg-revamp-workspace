# BATCH-051 — DEFINITIONS EXTRACT

## Terminology and Meanings

### Sleeve
Top-level identity/configuration artifact defining who the system is right now. In this batch, sleeves are used as authored gifts/identity artifacts, not only operational presets.

### Governance Anchor
The broader inherited control layer that may remain active even when not fully re-expressed inside a locally authored sleeve.

### Governance Expression
The explicit governance language embedded inside a sleeve artifact. This batch questions whether every sleeve should carry all enforcement language locally.

### Mythos / Identity Layer
Symbolic, creed-like, or relational expression of sleeve identity. In this batch, mythos is foregrounded in one revised sleeve draft after hard enforcement language is removed from the local sleeve body.

### NeoStack
Domain-level aggregation of NeoBlocks. In the Fractal Listener sleeve, stacks include resonance, fractal perception, quiet illumination, and heart/mind synthesis.

### NeoBlock
Compiled active/grouped intent unit inside a stack. In this batch, stacks expand into primary plus layered directive/instruction/philosophy-style blocks.

### MOLT Fields
Canonical atomic authoring fields used in expansions:
- Trigger
- Directive
- Instruction
- Subject
- Primary
- Philosophy
- Blueprint

### Blueprint / Rendering Layer
Output and packaging surface independent from cognition. Here, NL code block output and QR-linked artifacts are examples.
