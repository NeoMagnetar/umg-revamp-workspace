# BATCH-157 — Definitions Extract

## Working Definitions Surfaced in Source
- **UMG** = orchestration layer / modular thought architecture / systems wrapper
- **HRM** = compact latent reasoner / solver architecture
- **Directive / Blueprint** = planning / strategy overlays that choose route or method
- **Primary / Subject** = execution blocks or task-facing execution layer
- **Verifier** = validation layer for confidence checks, self-consistency, majority vote, or bounded retries
- **`hrm.solver`** = proposed wrapped UMG module name for ARC-like or grid-logic tasks
- **Explainability shim** = wrapper output exposing halt decisions, loop counts, and vote tallies around a latent solver
- **Guarded deployment** = wrapper pattern using confidence gates, cost caps, bounded retries, and fallbacks
- **Hybrid voters** = combined solver-internal voting plus UMG-level self-consistency or meta-grading
- **Operator surface** = the actual interface appropriate for execution work, distinguished here from a voice-helper overlay

## Definition Quality Notes
- These are source-faithful working definitions only.
- None of these terms should be treated as canon from this batch alone.
- `hrm.solver` remains proposed naming, not ratified naming.
- block / nCube / solver module remain unresolved alternatives around the same wrapped concept.
