# BATCH-140 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived runtime-pipeline summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Pipeline Terms
**Mind Spine**  
Layered structural representation of persona/cognitive architecture across identity, instruction, trigger, memory, and output surfaces.

**Bot Creation Pipeline**  
Process pattern for building deployable systems from structured identity definitions.

**Config-Driven Persona Injection**  
Runtime architecture in which persona/behavior is loaded from JSON or config structures rather than hardcoded prompts.

## Artifact Terms
**Identity Block Set**  
Root layer containing identity, values, and philosophy.

**Instruction Layer**  
Teaching, communication, coaching, or other procedural behavior layer.

**Snap Block**  
Triggered behavior block that activates under specific user-state or cue conditions.

**Strategy Block**  
Conditional higher-order response mode selected based on intent or context.

**Merge Block**  
Contextual modifier layer intended to blend or modulate adjacent behavior blocks.

## Deployment Terms
**Deployable Bot Package**  
Bundled artifact including configuration, runtime code, and execution environment setup.

**Resleeving by Config Swap**  
Implicit pattern in which a new identity/config package can generate a different bot without rewriting the whole runtime.
