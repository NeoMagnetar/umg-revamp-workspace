# BATCH-170 Definitions Extract

## Shared Block Anatomy
A proposed common anatomy for UMG blocks:
- `id`
- `name`
- `molt_type`
- `purpose`
- `content`
- `inputs`
- `outputs`
- `overlays`
- `merge`
- `snap`
- `metadata.ledger`

## Role Definitions Preserved
### PRIMARY
Core intent or anchor objective.

### SUBJECT
Domain, scope, or contextual frame.

### INSTRUCTION
Explicit rules, directives, guardrails, formatting commands, or execution guidance.

### PHILOSOPHY
Ethos, values, tone, or worldview layer.

### BLUEPRINT
Style, structure, or output-shaping layer.

### DIRECTIVE
Discussed as either separate from or adjacent to Instruction; unresolved.

### MERGE
Either a named block type or the general engine process that resolves block interaction; unresolved.

### ALIGNMENT
Policy layer capable of vetoing or overriding ordinary content behavior.

### PRIVACY
System-level control layer over data handling and privacy modes.

### FAILSAFE
Recovery logic when retrieval or LLM execution fails.

### ROUTING POLICY
Model/role selection logic.

### EXECUTION MODE
Runtime environment selector such as local/cloud mode.

### DEPLOYMENT
Packaging/runtime configuration layer.

### LIBRARY BLOCK
Reusable bundled composite of multiple blocks.

### MPC VOTE
Consensus/oversight mechanism for sensitive decisions.

## Unresolved Distinctions
- Directive vs Instruction
- Philosophy vs Blueprint
- Merge block vs merge engine logic
- dormant/experimental blocks vs active canonical blocks
