## Core Terms
### UMG
- Modular thought architecture using blocks/MOLT units.
- Intended to support structured logic, stacking, merging, and updating.

### MOLT
- Named block structure / taxonomy surface.
- Core fields named in the batch:
  - Primary
  - Subject
  - Instruction
  - Philosophy

### Block
- Logic unit within the UMG system.
- Can be stacked, merged, updated, and prioritized.

### Vault
- JSON file bundle used as persistent externalized cognition.
- Stores reusable blocks and structured logic outside the live chat context.

### CantoCore
- Internal compressed logic dialect.
- Used as a runtime-efficient internal representation rather than default human-readable prose.

## Runtime / Architecture Language
### Stack
- Ordered set of MOLT blocks loaded at runtime.

### Bundle
- Vault file or grouped block package used for loading/runtime composition.

### Merge
- Conditional recombination/update behavior between blocks or logic units.

### Priority
- Weighting logic influenced by recency/frequency.

### Self-Update
- Reflective update behavior gated by evaluation and user-controlled persistence.

### Resleeving
- Context change by loading different vault sets/project folders into a new modular instance.
