# BATCH-014 — Definitions Extract

## Terminology
### Block
- Atomic unit of meaning.
- Cognition-oriented rather than execution-oriented.
- Identity-stable across composition.

### NeoBlock
- Compiled active view of blocks within a stack.
- Supports one Primary and layered Instructions per stack context.
- May be shaped by bundle-driven activation / ordering.

### NeoStack
- Domain-level aggregation of NeoBlocks.
- Represents a mode of cognition or work.
- Explicitly activated; may be ON or OFF.

### Sleeve
- Defines who the system is right now.
- Contains governance, active stacks, defaults, and constraints.

### Governance Anchor
- Surfaced base anchor: `POE.UMG.CORE`.
- Tied to identity, signature, access, failsafe, and conflict order.
- Protected from mutation in runtime improvement logic.

### Bundle
- Groups blocks for ordering or alternation.
- In runtime materials also acts as a policy container.

### Merge
- Combines compatible blocks while preserving identity.
- Deterministic behavior required.

## Architecture / Runtime Language
### MODE vs BP_MODE
- MODE selects how the system thinks.
- BP_MODE selects how output is emitted.
- Orthogonal separation remains a core rule.

### Mode Activation Protocol
- Triggered by upload or request.
- Enumerate NeoStacks.
- Generate detailed NeoBlocks.
- Emit full sleeve structure in first response.
- Preserve deterministic naming integrity.
