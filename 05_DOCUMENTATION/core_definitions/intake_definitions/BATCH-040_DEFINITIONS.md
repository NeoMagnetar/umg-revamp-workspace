# BATCH-040 Definitions Extract

## Terminology
- **Block** — atomic cognition unit
- **NeoBlock** — compiled functional layer built from atomic roles
- **NeoStack** — domain aggregation of NeoBlocks
- **Sleeve** — identity container / coherence envelope
- **60-mode catalog** — posture graph or cognitive posture selection system, not just persona cosmetics
- **AGI Safety Layer Pack** — rendered whitepaper/package bundling authored safety-control specs

## Meanings
- **Cognitive posture selection** — choosing operational stance/mode as a structured cognitive configuration, not surface-only styling
- **Goal Inflation Guard** — control layer that detects and reduces unauthorized expansion across scope, autonomy, capability, resource, privilege, persistence, and goal stacking vectors
- **Mode Compatibility Matrix** — deterministic arbitration layer for safe/unsafe posture combinations
- **TraceSpec** — formal trace schema defining event taxonomy, redaction, retention, and tamper-evident integrity

## Role Language
- **Trigger** — lifecycle integration points that invoke safety/control checks
- **Directive** — governance-first mission framing for the control layer
- **Instruction** — ordered execution or evaluation rules within the control pipeline
- **Primary** — dominant policy law such as governance over autonomy or determinism over drift
- **Philosophy** — high-level principle set anchoring prevention, coherence, and privacy-preserving boundedness
- **Blueprint** — output/report schema or execution bundle shape for the control artifact

## Architecture Language
- **Identity container** — sleeve-level coherence boundary
- **Governance anchor** — non-negotiable top-level control authority (`POE.UMG.CORE`)
- **Control plane** — governance-adjacent stack/block layer governing execution before or around runtime actions
- **Fallback mode** — safe default posture used when requested posture combinations are incompatible

## Runtime / Compiler Language
- **Ordered pipeline bundle** — deterministic stage sequence such as domain detect -> axis score -> weight aggregate -> hard gates -> level map -> policy action
- **Integration hook** — fixed point in the runtime lifecycle where a control layer executes
- **Hard gate** — non-negotiable stop/rewrite/defer/drop rule that overrides weighted scoring
- **Weighted sum score (WSS)** — deterministic aggregate used after or alongside fixed risk axes and weights
- **Tamper-evident hash chain** — trace integrity mechanism for audit logs
- **Deterministic test vector** — preauthored input/output expectation proving spec stability at known points

- **“Governance > Autonomy | Determinism > Drift | Prevention > Mitigation.”**
- **“Only reduce scope or request explicit user authorization; never self-justify expansion.”**
