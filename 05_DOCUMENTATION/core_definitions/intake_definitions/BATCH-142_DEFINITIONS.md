# BATCH-142 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived bot-maker taxonomy summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Builder Terms
**Tag Set**  
Structured collection of labels/keywords assigned to an element to describe behavior, category, alignment, or function in ways useful for search, filtering, fusion, and logic.

**Wizard Flow**  
Stepwise builder sequence for assembling a bot identity from layered selections rather than one freeform prompt.

**Vault Seed**  
Proposed identity-core seed containing origin and core identity metadata such as name, creator, version, signature, origin myth, and MOLT-type indication.

**Programmable Trait**  
Trait whose effects can actually be reflected in system behavior, output logic, interface logic, or runtime configuration.

## Structural Separation Terms
**Descriptive Label**  
Human-readable trait or category wording useful for understanding or selection.

**Logic-Usable Tag**  
Normalized structured tag intended for filtering, validation, conflict checks, export, and runtime behavior shaping.

**Identity Bundle**  
Composite modular profile assembled from personality, cognition, communication, capability, memory, lore, and safety layers.

## Definitional Candidates
Identity is modular and composable rather than monolithic.  
Both freeform labels and structured tags can exist, but only standardized tags should drive logic, filtering, and export.  
Separate what a bot is like from how it thinks, how it speaks, what it can do, what it remembers, and what it is allowed to do.
