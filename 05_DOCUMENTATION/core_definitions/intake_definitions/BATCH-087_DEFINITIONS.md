# BATCH-087 — Definitions Extract

## Terminology
### UMG
Modular cognitive architecture / composable operating system for synthetic minds.

### MOLT
Structured syntax for logic and behavior; modular thought unit within UMG.

### Belief-Based Financing
External financing posture where support is tied to conviction in UMG’s success rather than equity ownership.

### Honor Clause
Trust-based governance framing used inside the financing layer.

## Role Language
### Primary
Main directive or objective.

### Subject
Domain orientation.

### Philosophy
Tone, worldview, or value lens.

### Instruction
Rules, constraints, or execution controls.

### Blueprint
Identity / scope / presentation layer.

### Merge
Additive content integration layer for support information or constraints.

### Instruction Node
Filter / format / rule application layer.

## Runtime / Compiler Language
### Merge Engine
System component that merges blocks into prompt stack and handles hierarchy/execution order.

### Token Weight
Relative influence assigned to a block during execution.

### Priority
Execution-order weighting applied across blocks.

### Pre/Post Pipeline
Pre-response build followed by post-response adjustment.

- UMG = composable operating system for synthetic minds
- blocks of thought as primitives
- Primary defines objective, Merge injects detail, Blueprint shapes output
