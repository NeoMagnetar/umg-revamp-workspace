# BATCH-066 Definitions Extract

## Terminology surfaced
### Immutable Core
A central non-trivial kernel for a deployable UMG agent that preserves alignment, risk constraints, and core action rules while allowing outer-layer adjustments.

### Philosophy Overlay
A style/strategy layer placed above the core kernel to alter trader stance or reasoning approach without replacing the kernel.

### Instruction MOLT
A modular operational rule block used for risk limits, entry logic, exit logic, reaction rules, or execution constraints.

### Metamolt
A micro-adjustment layer for subtle runtime tuning such as temporal bias, sector bias, sentiment bias, or pattern recall emphasis.

### Shared Vault
A coordinated memory/state surface used by multiple specialized agents for conflict logging, shared context, and synchronization.

### Trifecta
A tethered three-agent model with differentiated roles and shared-vault coordination.

### Async Convergence
A coordination mode where multiple agents contribute asynchronously and converge on a decision under logging and gating rules.

### Confidence Threshold Execution
A bounded execution rule where action is allowed only if confidence exceeds a defined threshold.

### Rendering Separation
The distinction between the underlying block logic and the artifact/display mode used to present it, such as QR, plaintext CantoCore, JSON, or image blocks.
