# BATCH-082 — Definitions Extract

## Terminology
### UMG
Underlying architecture or scaffolding for modular cognition and output composition.

### MOLT
User-facing compositional grammar or operating language layered on top of UMG.

## Role Language
### Template / Primary
Top-level directive or main task; what to do.

### Subject
Topic, domain, or what the construct is about.

### Philosophy
Lens, tone, worldview, or how to think.

### Instruction
Constraint, rule, or behavioral control; how to behave.

## Builder Language
### Snap
User action or system behavior that connects blocks into hierarchy or structural relation.

### Stack
Top-down structure expressing inheritance, context, or scope.

### Merge
Fusion of logic, styles, rules, or philosophies into composite cognitive units.

### Skeleton Mode
Structured outline / nested cognitive tree view for organizing relationships among blocks.

### Preset / Example Pack
Curated reusable starting content for a given block type.

### Block Database
Future searchable repository of reusable blocks and templates.

## Architecture Language
### Grammar vs Architecture
UMG provides the structural architecture; MOLT provides the compositional grammar visible to users.

### Branch-Scoped Logic
Philosophy or instruction applied only to specific sub-sections or branches rather than to the full construct.

## Definitional Candidates
- UMG is the architecture; MOLT is the grammar.
- MOLT = Modular Operating Language of Thought.
- Template / Subject / Philosophy / Instruction = the plain-language role set for user-facing composition.
