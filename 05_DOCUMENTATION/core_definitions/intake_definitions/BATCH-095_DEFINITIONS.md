# BATCH-095_DEFINITIONS

## Scope Note
These definitions are extracted from a derived source summary. They are evidence candidates, not canon.

## Terminology Extract

### Shape brain
- Informal explanation layer for a specialized subsystem focused on geometry, spatial structure, or non-linguistic representation.
- Functions as a modular intelligence component rather than a whole system.

### Language brain
- Informal explanation layer for a subsystem focused on reasoning and text generation.
- Paired with the shape brain in the source as part of a composable architecture.

### Geometry encoder
- Candidate specialized processing block.
- Handles spatial or shape-structured data.
- Open question: documentation analogy only, or reusable NeoBlock pattern.

### Adapter / Fusion layer
- Bridge mechanism connecting specialized representations to language/reasoning output.
- High-value candidate for future interoperability or bridge-block semantics.

### Modular recomposition
- Runtime rearrangement or assembly of system parts according to task.
- Presented as valuable but stability-sensitive.

### Bounded recomposition
- Constraint concept implying that dynamic assembly or self-modification requires orchestration and control surfaces.

## Role Language

### Subject
- implied domain specialization such as geometry, shape, or space

### Primary
- implied reasoning/output generation role

### Instruction
- implied bridge or mapping behavior between representational domains

### Philosophy
- modularity, composability, and structured intelligence over monolithic architecture

## Architecture Language

### Specialized block
- Processing unit optimized for one kind of representation or task
- candidate examples: geometry encoder, language reasoning model

### Bridge block
- Block-like interoperability mechanism allowing specialized units to exchange usable representations
- candidate example: adapter / fusion layer

### Dynamic stack assembly
- Task-dependent construction of a stack or graph from multiple specialized components

### Micro-block modification
- Small tuning or behavior-shaping units such as LoRA-like modules
- may behave like local modifiers rather than large standalone blocks

## Runtime / Compiler Language

### Runtime orchestration
- Selection and assembly of active modules according to task

### Translation boundary
- Interface where one internal representation is converted into another usable representation for downstream reasoning or output

### Stability/control layer
- Governance-like mechanism needed to keep recomposition bounded and traceable
