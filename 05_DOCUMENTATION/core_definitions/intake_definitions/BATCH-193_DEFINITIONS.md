# BATCH-193 - Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived builder-system summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Terms
Schema-Driven Block
A block whose rendering and interactions are driven from an underlying structured schema rather than from hardcoded ad hoc UI.

Inspector Grid
Structured inspection/edit surface for block internals such as identity, metadata, advanced settings, code modules, and connections.

Drop-to-Block
Candidate authoring workflow in which imported information is parsed and compiled into a labeled canonical block.

Render Skin
Optional visual geometry/theme layer such as rectangle, puzzle, cog, or hex, applied over stable underlying block semantics.

## Definitional Candidates
- Blocks should carry MOLT identity, code/text content, Cantocore metadata, merge logic, and connection behavior.
- The UI should be driven from a canonical block schema rather than hardcoded cards.
- Block semantics should remain stable while surface geometry and layout modes vary.
