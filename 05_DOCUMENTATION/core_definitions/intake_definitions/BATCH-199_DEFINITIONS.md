# BATCH-199 — Definitions Extract

## Terminology
### Cognitive Specification Layer (CSL)
Tech-agnostic layer specifying what cognition should exist rather than how runtime execution occurs.

### RuntimeSpec
Structured, execution-neutral output of UMG compilation. Not itself a prompt.

### Trace
Mandatory chronological record of compile actions and resolutions. No Trace = not UMG-compliant.

### Sleeve
Static, inert container of potential cognition containing blocks, snaps, defaults, domains, and metadata.

### Block
Atomic unit of cognition. One idea / rule / value / constraint. Inert until composed and compiled.

## MOLT Types
- Trigger
- Directive
- Instruction
- Subject
- Primary
- Philosophy
- Blueprint
- Off

## Role Language
### PrimaryShell
Primary output-bearing role; distinct from MOLT type Primary.

### MergeContributor
Role for compatible content participating in merge operation.

### BlueprintGuide
Silent structural/style influence role.

## Architecture / Mechanics
### Stack
Strict same-type priority resolution mechanism at compiler level.

### Merge
Operation for combining compatible same-type content with a shared non-empty merge_key.

### Domain
Favored human-facing candidate term for cross-type conceptual grouping.

### Snap
Scope/membership selection surface used during compilation.

## Definitional Candidates
- Universal Modular Generation (UMG) is a tech-agnostic Cognitive Specification Layer (CSL).
- MOLT is the formal grammar of UMG.
- MOLT Primary != Role PrimaryShell.
- UMG specifies cognition; execution is external.
- Sleeves are lifeless containers of potential cognition.
