# BATCH-114 — Definitions Extract

## High-Confidence Definition Candidates
### UMG
A cognitive specification and synthesis system whose purpose is to make cognition explicit, structured, constrained, and auditable prior to execution.

### Core Principle
If something matters, it must be explicit, inspectable, and traceable.

### Governance
The supreme participation constraint answering one question: **What is permitted?**

### OFF
The only exclusion mechanism.

### Trigger
An eligibility gate answering **When eligible**. It adds no meaning, no authority, and does not decide outcomes.

### Priority
A deterministic, local conflict-resolution mechanism that matters only at conflict sites and carries no semantic weight beyond that role.

### Merge
The only mechanism by which meaning fuses. It is explicit, traceable, creates a new block, and does not mutate source blocks.

### Bundle
A horizontal structural grouping that creates no new authority and no new meaning.

## MOLT Role-to-Question Mapping
- **Primary** → Why
- **Subject** → About what
- **Directive** → How (strategy)
- **Instruction** → Under what rules
- **Philosophy** → Under what values
- **Blueprint** → In what form
- **Trigger** → When eligible

## Structural / Boundary Definitions
### Vertical
Authority / constraint

### Horizontal
Coexistence

### CSL Boundary
UMG operates inside CSL. Execution always happens outside CSL.

### Determinism Condition
Given identical blocks, structure, governance, triggers, and priorities, the result must be identical.

### Auditability Condition
Every outcome must explain what participated, what was excluded, and why selection occurred.
