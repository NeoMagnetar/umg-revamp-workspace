# BATCH-083 — Definitions Extract

## Terminology
### UMG
Modular cognition framework / cognitive architecture built from swappable or composable blocks.

### NeoUMG
Public-facing or document-specific naming variant used across whitepapers and release materials.

### MOLT
Modular Operating Language of Thought; atomic thought unit or modular logic unit, described as editable, stackable, and mergeable.

## Role Language
### Primary
Central directive or core behavior.

### Subject
Theme, topic, or domain.

### Philosophy
Lens, worldview, logic directive, or alignment framing.

### Instruction
Operational rule, behavioral constraint, or execution rule.

### Blueprint
Intent/style/structure layer shaping output without necessarily inserting literal content.

### Merge
Logic fusion or support-content integration layer that conditionally combines multiple blocks.

## Release / Legal Language
### Apache 2.0
Actual repo license clarified in the batch.

### Public Prior Art
Publication strategy used to create timestamped public disclosure and reduce later exclusivity claims by others.

### Patent Pending
Proposed wording discussed in the batch, but not consistently supported by actual filing state in the source.

## Runtime Language
### Priority Toggle
0/1/2/3 execution state system affecting token weight and execution order.

### Token Weight
Relative execution weighting used in the Commando runtime model.

### UMG Preamble
Instructional system layer injected before model execution in the Commando concept.

## Definitional Candidates
- UMG is a modular system for building artificial intelligence through swappable cognitive blocks.
- MOLT is the atomic unit of thought — editable, stackable, mergeable.
- Merge Block = logic fusion that conditionally combines and prioritizes multiple MOLTs.
