# BATCH-006 — Definitions Extract
## Terminology
### Block
- Atomic unit of meaning with id, role, content, and tags.
- Semantic unit that describes cognition rather than execution.

### Sleeve
- What the system is right now.
- Top-level runtime identity containing governance, active stacks, defaults, and constraints.

### NeoBlock
- Compiled active view of blocks within a stack.
- Should eventually reflect resolved active truth rather than raw concatenation alone.

### NeoStack
- Domain-level aggregation of NeoBlocks.
- Explicitly activatable runtime grouping rather than passive labeling.

### Authority Resolution
- Proposed compiler/runtime phase where MOLT decides what is actually true before prompt assembly decides how to say it.
- Intended bridge between normalization and output construction.

### Prompt Assembly
- Downstream emission/layout phase.
- Distinct from authority and conflict resolution.

### Tag
- Current index surface used for lookup/grouping.
- Not yet equivalent to runtime signal or condition evaluation.

### Signal / Condition
- Candidate runtime surfaces for turning observed state into branch or winner selection.
- Distinguished from plain static tags in this batch.

## Role Language
- **TR / Trigger** — event initiator or branch activator
- **DIR / Directive** — strategic or behavior mode
- **INS / Instruction** — operational rule
- **SUB / Subject** — domain/context lens
- **PRI / Primary** — anchor/identity/stable core
- **PHI / Philosophy** — worldview or additive lens
- **BP / Blueprint** — presentation/output shaping
