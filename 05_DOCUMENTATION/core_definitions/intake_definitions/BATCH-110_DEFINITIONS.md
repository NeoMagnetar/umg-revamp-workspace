# BATCH-110 — Definitions Extract

## Terminology Status
This batch is unusually definition-heavy. The source presents these items as high-value terminology candidates, but not all should be treated as final canon without explicit project approval.

## Core Definitions
- **UMG**
  - Formal cognitive specification system
  - Makes cognition explicit, constrained, auditable, and deterministic prior to execution
  - Specifies cognition; does not execute behavior

- **Cognition**
  - First-class, inspectable artifact
  - Not an implicit byproduct of prompts or execution

- **Governance**
  - Supreme supervisory layer
  - Applies before authority or priority resolution
  - May exclude cognition entirely

- **Deterministic Resolution**
  - Same inputs must yield same resolved results

- **Auditability**
  - Resolution must explain participation, exclusion, and conflict handling

## MOLT / Role Language
- **MOLT**
  - Typed control grammar / semantic role grammar
  - Preserves type-level structure while academic-safe language remaps wording
- **Open issue**
  - Exact ordering and rigidity of MOLT block types explicitly remained unresolved

## Neo-Structure Language
- **NeoBlock**
  - Synthesized block formed from multiple resolved blocks
  - Academic-safe mapping offered: “Derived Control Primitive”
  - Partial stability only; full synthesis rules remain open

- **NeoStack**
  - Composed structure of synthesized cognitive blocks treated as a single cognitive unit
  - Academic-safe mapping offered: “Composite Control Structure”
  - Authority semantics remain unresolved

## Runtime / Compiler Language
- **CIR**
  - Execution-facing structured representation / compiled output of resolved cognition
  - Must not be described as adding meaning during compilation

- **RuntimeSpec**
  - Bounded specification that governs execution-time reasoning constraints and intent

- **Trace**
  - Structured audit record of participation, suppression, and conflict resolution

- **PCL**
  - Operational/runtime interpretation layer applying resolved cognition during execution
  - Does not author cognition

## Documentation Packaging Language

- **Mappings**
  - Reversible terminology translations
- **Proof Language**
  - Formal semantics framing showing equivalence without renaming meaning
- **Open Questions / Canon Delta**
  - Explicit unresolved issues tracked without silent canonization
