# BATCH-086 — Definitions Extract

## Terminology
### UMG
Modular cognitive architecture using composable blocks.

### NeoUMG
Repo/system framing for the broader modular AI framework deployment.

### NeoCPF
Provenance / traceability / authorship layer intended to extend UMG with auditability and origin tracking.

## Role Language
### Primary
Objective-defining block.

### Subject
Domain / context block.

### Philosophy
Tone / value / worldview block.

### Instruction
Behavioral or execution-constraint block.

### Merge
Support-logic injection layer for auxiliary data or details.

### Blueprint
Style / output-shaping layer separate from direct content injection.

## Governance / Release Language
### Apache 2.0
Chosen repo license; permissive but attribution-preserving.

### NOTICE
File asserting authorship and Apache terms in public release structure.

### CONTRIBUTING
Contribution-governance surface defining modification and participation flow.

### SAFE Agreement
Funding / ownership structure artifact adjacent to commercialization posture.

## Definitional Candidates
- UMG is a modular cognitive architecture using composable blocks.
- Primary defines objective, Merge injects detail, Blueprint shapes output.
- NeoCPF = provenance layer for traceable cognition and authorship.
