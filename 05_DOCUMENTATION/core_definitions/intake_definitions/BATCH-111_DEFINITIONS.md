# BATCH-111 — Definitions Extract

## Terminology, Meanings, and Candidate Definitions

### Google Drive role
- Proposed as **canonical narrative memory / source corpus**.
- Holds human-readable UMG material such as theory, architecture, warnings, and implementation scaffolding.
- Should be treated as evidence input until canonized.

### Structured registry role
- Proposed as **structured operational truth / ledger**.
- Intended to hold tables or records for blocks, sleeves, governance rules, RuntimeSpec, compiler requirements, and trace events.
- Registry structure is proposed, not finalized in this batch.

### GitHub role
- Proposed as **source code / compiler / repo** surface.
- Intended to hold compiler code, docs, and deployment-bound code assets.

### ChatGPT role
- Proposed as **orchestration bridge / conductor layer**.
- Bridges tools conceptually rather than apps merging directly with each other.
- Operates in a human-in-the-loop mode with the user as governor / architect.

### Primary
- Surfaced framework snippet meaning: base abilities / ethical compass.

### Merge
- Surfaced framework snippet meaning: contextual adaptation or injected information.
- Brought in in a controlled way so core logic is preserved while task-specific information is blended.

### Blueprint
- Surfaced framework snippet meaning: high-level plan, tone, philosophy, or response feel.

### Trigger
- Present in warning vocabulary and diagram proposals.
- Not materially extended in this batch beyond diagnostic/runtime warning usage.

### Priority
- Present in warning vocabulary as conflict-resolution selection logic.

### Governance
- Present through warning/error language and sidecar/gateway safety framing.
- Treated as part of runtime control, conflict handling, and guardrail enforcement.

### TraceEvent
- Example runtime/compile diagnostic record with severity, code, message, related block IDs, and governance rule IDs.

### CompileResult
- Example compile structure including runtime, trace, and error state.
