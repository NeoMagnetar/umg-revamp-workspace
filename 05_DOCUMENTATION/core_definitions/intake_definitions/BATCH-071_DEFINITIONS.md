## Terminology and Meanings

### UMG
- Modular cognitive architecture built from composable blocks.
- Scaffold for cognition rather than a single monolithic prompt.

### MOLT Roles Referenced
- **Primary** — what to do.
- **Subject** — domain or topic focus.
- **Instruction** — how to do it.
- **Philosophy** — lens or worldview.

### Alignment
- Proposed as a governing validity layer.
- Framed as “why allowed” or what makes execution acceptable.
- Suggested to enforce ethical/value coherence and prevent drift.
- Canon status unresolved.

### Merge
- Combines multiple blocks into a unified output.
- Must preserve coherence across blocks and role hierarchy.

### Blueprint
- Presentation-shaping layer for style, structure, or rendering behavior.
- Separated from core content-generation roles.

### Stack Builder
- Proposed interface for composing visible UMG stacks from blocks.

### Alignment Studio
- Proposed interface for editing, testing, and diagnosing alignment behavior.

### Drift Detection
- Proposed mechanism for identifying deviation from intended values or constraints.

### Execution Gating
- Proposed enforcement behavior where misaligned stacks may be halted or corrected before execution.
