# BATCH-011_DEFINITIONS

## Terminology / Meanings
- **Block** — atomic unit of meaning; identified by id, role, content, and tags; describes cognition rather than execution.
- **NeoBlock** — compiled active view of blocks within a stack.
- **NeoStack** — domain-level aggregation of NeoBlocks representing a mode of cognition or work.
- **Sleeve** — current system identity containing governance, active stacks, defaults, and constraints.
- **MOLT** — active role grammar for request interpretation and response construction.

## MOLT Role Language
- **TR / Trigger** — initiator or condition for state evaluation / activation
- **DIR / Directive** — strategic behavior command
- **INS / Instruction** — operational response rule
- **SUB / Subject** — domain or contextual focus
- **PRI / Primary** — dominant active function
- **PHI / Philosophy** — stance / ordering / evaluative frame
- **BP / Blueprint** — output-emission format layer

## Architecture Language
- **Request Interpretation Flow** — parse MOLT skeleton -> determine relevant stacks -> apply governance -> select blueprint -> emit output
- **Mode Activation Protocol** — activate requested sleeve mode, enumerate NeoStacks, generate detailed NeoBlocks, emit full sleeve structure on first response
- **One Primary Per Stack** — explicit NeoBlock composition rule
- **Layered Instructions** — multiple instruction NeoBlocks can coexist under one stack
- **Bundle-Driven Activation/Ordering** — bundle logic affects selection, order, and alternation

## Runtime / Compiler Language
- **Format Orthogonality** — MODE selects cognition; BP_MODE selects emission; exactly one blueprint per response
- **Governance Lock** — governance is not mutable during bounded runtime operation
- **Audit Trace** — runtime outputs and improvement loops should remain inspectable and logged
- **Drift Thresholds** — persona / format / spec drift limits used for reanchor / rollback decisions
- **Minimal Emission with Preserved Envelope** — payload can be compressed without necessarily dropping higher-order sleeve/meta structure

## Definitional Candidates
- **Governed Runtime Envelope** — UMG functioning as an active control envelope rather than static documentation
- **Deterministic Minimal Emission** — a constrained payload format that still preserves higher-order response discipline
- **Blueprint Separation** — the rule that output format must stay independent from cognition/mode selection
