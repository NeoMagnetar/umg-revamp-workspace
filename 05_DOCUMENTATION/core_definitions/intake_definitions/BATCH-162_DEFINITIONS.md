# BATCH-162 — Definitions Extract

## Poe
One preserved underlying identity that may operate across multiple runtime surfaces.

## Sleeve
A surface-specific runtime embodiment of the same core identity. In this batch, sleeves were proposed as CLI Poe, web Poe, and assistant/dev Poe.

## Resleeving
Moving the same core identity into a different runtime body or interface surface without redefining the identity itself.

## Shared Core
Reusable cognition/decision logic imported by sleeves rather than reimplemented in each runtime file.

## CLI Sleeve
A Poe runtime surface intended for terminal interaction and local prompt flow.

## Web Sleeve
A Poe runtime surface intended for browser/server interaction, typically through an exported request handler.

## Assistant Sleeve
A future background/dev-oriented Poe surface intended for non-CLI, non-web operational contexts.

## Output/Rendering Separation
The distinction between shared cognition and interface-specific expression, such as terminal prompts versus web route handling.

## Import-Safe Runtime
A runtime file that can be imported without immediately auto-executing terminal or blocking logic.

## Tarball Handoff
A bounded delivery method where modified/new files are bundled locally for manual extraction rather than pushed remotely.
