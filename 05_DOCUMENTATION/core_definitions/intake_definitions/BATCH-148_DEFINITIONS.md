# BATCH-148 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived large-scale builder summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core System Terms
**Snap Logic Engine**  
Composition layer described as constructing the final chatbot stack by merging MOLTs, resolving conflicts, and producing the active configuration.

**MOLT Builder**  
Assembly layer that selects and stacks MOLT-bearing blocks into the final chatbot structure.

**Realtime Snap Preview**  
Live preview surface showing stack assembly or dependency consequences as configuration changes.

**MOLT Persistence Modes**  
Modes described for MOLT behavior/state persistence: session, memory, and profile.

## Builder Terms
**Full Chatbot Builder System**  
28-category, 280-block modular architecture for constructing chatbot identity, logic, interface, safety, deployment, and orchestration through standardized block artifacts.

**Suggested MOLT Merges**  
Explicit merge recommendations for composing persona, tone, profession, quirk, philosophy, and instruction layers.

**Instruction Injection**  
Temporary override or augmentation layer using merge policies such as override, append, or contextual blend.

## Definitional Candidates
Blocks are living modular intelligence units.  
Transition from ad-hoc prompts to structured block-based system.  
A complete chatbot is a stack assembled through MOLT Builder and Snap Logic.
