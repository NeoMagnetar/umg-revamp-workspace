# BATCH-129 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived positioning summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core System Terms
**UMG**  
Universal Modular Generation; a Cognitive Specification Layer that describes cognition structurally and deterministically.

**Cognitive Specification Layer (CSL)**  
A layer that specifies what cognition is rather than how it executes.

**RuntimeSpec**  
Compiled artifact describing what is active, constrained, allowed, or suppressed.

**Trace**  
Forensic record of how the final cognitive structure was derived.

**Block**  
Atomic unit of cognitive meaning.

**MOLT**  
Modular Operating Language of Thought; a typed semantic role system that defines ordering of influence.

**Governance**  
Explicit conflict-resolution layer and the only legitimate suppression or override mechanism.

**Sleeve**  
Static container holding blocks and stacks; lifeless until compiled.

**Compiler**  
Deterministic transformer that applies governance, ordering, and merges, then emits RuntimeSpec plus Trace.

## Proposed Extension Terms
**Hard Gates**  
Deterministic validity checks that cannot be negotiated away.

**Soft Scores**  
Rubric-based evaluations used to rank surviving candidates after hard validation.

**Trace-of-Selection**  
Proposed expanded trace that would log candidate generation, rejection reasons, scores, and final selection rationale.

## Definitional Candidates
UMG is a Cognitive Specification Layer.  
UMG is not an agent, not AGI, and not a runtime loop.  
UMG separates specification, compilation, and execution.  
UMG is closer to a compiler / IR / governance layer for cognition.
