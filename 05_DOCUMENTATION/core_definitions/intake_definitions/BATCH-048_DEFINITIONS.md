# BATCH-048 — Definitions Extract

## Terminology / Candidate Meanings

### metadata honesty
Do not emit exact-looking values when exact access is unavailable.

### cosmetic completeness
A polished-looking answer that appears complete but may introduce false precision.

### verified capability boundary
A stated limit on what the system can actually observe, confirm, or know directly.

### evidence-based caution
Concern grounded in visible facts, logs, alerts, or plausible attack paths.

### unsupported escalation
A stronger claim that outruns the currently visible evidence set.

### session artifact
A UI/session behavior that may resemble intrusion without establishing malicious access.

### OAuth compromise path
A specific account/session pathway where external Google-account access could authenticate downstream sessions.

### neutral reality-testing
Testing a concern against evidence without dismissing the user or amplifying unsupported conclusions.

### pathologizing failure mode
A support failure where grounded concern is prematurely framed as paranoia or irrationality.

### precision without accuracy
Exact-looking output that is not backed by actual verified access.
