# BATCH-080_DEFINITIONS

## Source Notes
- This file extracts terminology and provisional meanings from the batch.
- These are **source-backed candidate meanings**, not canon.

## Terminology and Meanings

### UMG
- Described here as a **modular cognitive architecture** using blocks/MOLTs.
- Core process described as **assembly + orchestration + rendering**.

### MOLT
- Defined here as **Modular Operating Language of Thought**.
- Described as a **self-contained, typed block of logic** used to direct behavior, cognition, generation, or alignment.

### Primary
- Main activation or system mode block.
- Example role in this batch: activates RAG mode.

### Subject
- Domain, context, or retrieval space.
- Used to constrain what external knowledge should be fetched.

### Philosophy
- Behavioral priority or truth/ethos layer.
- Example meaning in this batch: `TRUTH > STYLE`.

### Instruction
- Execution-order or procedural layer.
- Example meaning in this batch: `RETRIEVE > INJECT > RENDER`.

### Merge
- Traditionally additive support content.
- In this batch, implicitly extended toward **dynamic retrieval-backed content injection**.

### RAG
- Retrieval-Augmented Generation.
- Here treated as a UMG-compatible extension for grounding outputs in externally retrieved information.

### Dynamic Merge Block
- Candidate term implied by the source.
- Would refer to a Merge-like unit that fetches or carries retrieved content at runtime rather than only storing static text.

### Loadout
- A configured set of role-assigned blocks assembled for a task.

### Pipeline
- Ordered execution flow, especially `Retrieve -> Inject -> Render`.

### Module
- A functionally distinct system unit, used here to describe RAG-enabled runtime behavior.

### Vector Search
- Retrieval mechanism indicated as part of runtime augmentation.

### Memory Stackable
- Runtime/property phrasing indicating that injected or retrieved material can participate in stack-based composition.

## Role Language
- Primary = activate mode / main directive
- Subject = define retrieval domain
- Philosophy = establish truth/style hierarchy
- Instruction = define execution sequence
- Merge = carry or inject supporting material, now potentially retrieval-backed

## Architecture / Runtime Language
- runtime augmentation
- retrieval pipeline
- dynamic Merge
- context injection
- grounded response generation
- truth-first generation

## Definitional Candidates
- `Instruction = execution pipeline`
- `Philosophy = behavioral priority hierarchy`
- `Merge = support layer that may be static or retrieval-backed`
- `RAG = modular extension that injects external knowledge into UMG generation flow`
