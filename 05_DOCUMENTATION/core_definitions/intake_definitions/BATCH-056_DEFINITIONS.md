# BATCH-056 — Definitions Extract

## Terminology
### UMG
- Described in this batch as a modular reasoning framework and scaffold for composable intelligence.
- Positioned as a human-readable grammar for modular cognition rather than only a prompt format.

### NeoUMG
- Described as a more extended modular AI framing using swappable cognitive blocks intended to be interpretable, upgradable, intentional, and human-aligned.

### PoeUMG
- Agent protocol layer using MOLT-driven cognition, alignment controls, reflection, and modular control surfaces.

### Self-Adaptive UMG Agent
- Advanced agent layer introducing adaptation, user-style mimicry, behavior adjustment, and post-response evolution.

### NeoModGen
- README/project surface describing an early product/workflow split:
  - studio
  - engine
  - docs
  - templates

## Role Language
### Primary
- Central idea, dominant mode, or leading directive.

### Subject
- Domain or knowledge orientation.

### Philosophy
- Lens, tone, worldview, or logic filter.

### Instruction
- Operational rule or behavioral constraint.

### Merge
- Supporting content and/or logic fusion mechanism.
- Also used in advanced layers as governed behavior evolution.

### Blueprint
- Used in this batch across multiple scopes:
  - style/structure guide
  - identity + scope declaration
  - governance/policy influence

### Overlay
- Extra active stack layer in PoeUMG examples beyond the base MOLT type list.

### Meta Layer
- Runtime-relevant override, recursive logic, or conflict-resolution layer.

## Architecture Language
### Modular reasoning
- Claim that the system organizes reasoning structure, not just modular inputs.

### Composable intelligence
- Framing of UMG as a scaffold for assembling interpretable cognitive structures.

### Modular memory
- Memory handled through blocks/MOLTs rather than only transcript accumulation.

### Reflection injection
- Runtime insertion of reflective behavior after or during response construction.

## Runtime / Compiler Language
### Role-aware orchestration
- Engine behavior that treats different block types differently at runtime.

### Change detection
- Trigger or flagging mechanism when operational logic shifts.

### Post-response adjustment
- Runtime feedback loop that changes weights or block state after a response.

### Controlled mutation
- Governed, traceable block change rather than free mutation.

## Definitional Candidates
- UMG as modular reasoning rather than merely modular input.
- UMG as a scaffold for composable intelligence.
- MOLT as the atomic unit of thought in expanded UMG documentation layers.
- Base UMG, NeoUMG, PoeUMG, and Self-Adaptive UMG Agent as layered documentation strata rather than interchangeable labels.
