# BATCH-200 — Definitions Extract

## Terminology
### Modular Operating Language of Thought (MOLT)
Typed, swappable logic-block framework used to structure reasoning.

### Citadel Core / Orchestrator
Inferred coordinating layer that combines Primary, Merge, and Blueprint outputs into a final reasoning or prompt sequence.

### Vault / Memory
Inferred persistent or retrievable context source used by Merge-like logic.

### Architecture-Aware Reasoning
Agent capability to inspect and reason about system modules, documentation, and relationships rather than only manipulating text blindly.

## Role Language
### Primary Block
What you want; user intent or core prompt.

### Merge Block
What to include; auxiliary context or retrieved supporting information.

### Blueprint Block
How it should feel; style, format, tone, or response philosophy.

## Capability / Safety Language
### File-System Capability Layer
Explicit interface for listing, viewing, and editing files.

### Command / Execution Layer
Bounded interface for running or simulating commands.

### Propose-and-Confirm Workflow
Pattern where changes are surfaced as diffs or proposals before being applied.

### Sandbox Execution
Restricted environment for running code or commands with limits and controls.

## Definitional Candidates
- Primary = what you want
- Merge = what to include
- Blueprint = how it should feel
- architecture-aware reasoning support
- bounded coding-agent capability layer
