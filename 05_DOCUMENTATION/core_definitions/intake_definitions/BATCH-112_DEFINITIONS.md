# BATCH-112 — Definitions Extract

## Terminology surfaced in this batch

### System 1 (S1)
Fast, automatic, immediate cognition.

### System 1+ (S1+)
Guided or posture-shaped cognition that remains short of fully constructed deliberation. In this batch, it is associated with MOLT-based guidance but not full cognitive construction.

### System 2 (S2)
Classical deliberate reasoning.

### System 2′ (S2′)
Constructed, externalized, governed deliberation that exists prior to execution and is inspectable, reusable, and portable.

### CIR
**Cognitive Intermediate Representation.** Introduced here as a candidate representation/object language for S2′ and as a substrate that can compile into execution plans for different runtimes.

### Governance
Upstream constraint/control layer. In this batch, governance is described as preceding inference and shaping what deliberation is allowed.

### Execution Plan
Substrate-specific plan produced from CIR for humans, LLMs, tools, or organizations.

### MOLT
Referenced here primarily as a guidance grammar relevant to S1+ rather than as a fully reworked taxonomy in this specific chat.

## Role language and architecture language

### Pre-execution cognition
Cognition is built before runtime rather than emerging only at runtime.

### Cognition as infrastructure
Cognition is treated as a reusable structured artifact, not merely as a transient process.

### Inspectability
Constructed cognition can be reviewed, audited, and validated.

### Portability
The same cognitive artifact can be compiled to multiple runtimes/substrates.

### Governance-first
Constraints, permissions, and policy shape deliberation before execution.

## Definitional candidates
- “System 2 reasons; System 2′ constructs reasoning.”
- “Cognition as infrastructure.”
- “S2′ is the regime; CIR is its object language.”
- “Constructed deliberation (pre-execution).”

## Unresolved definitional areas

- CIR ↔ block/stack/NeoBlock relation
