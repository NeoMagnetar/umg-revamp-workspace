# BATCH-062 Definitions Extract

## Terminology
### UMG
A modular cognitive architecture for building outputs and behavior through structured blocks rather than one monolithic prompt.

### Primary
Core objective or main directive.

### Merge
Supporting information, features, constraints, or data integrated into a primary task.

### Blueprint
Tone, style, philosophy, or structure guidance that shapes output without acting as literal injected content.

### MOLT
Modular operating / thought unit used to shape behavior, strategy, reflection, memory, and runtime control.

### Vault Header
Boot-chain / initialization layer carrying system identity, deployment signature, access mode, and failsafe-style governance fields.

### Alignment Block
Identity, philosophy, moral logic, and behavioral constraint layer that regulates downstream behavior.

### Overlay Block
Context-specific modulation layer that adjusts runtime behavior without replacing the core aligned identity.

### MetaMOLT / Meta Layer
Override / recursive logic layer used for checklist execution, mutation monitoring, reflective logic, or memory maintenance.

### Trigger Block
Command- or state-driven runtime action such as save, sync, reduce, check, prune, or rewrite.

### Memory Sweep Logic
Prune unused tags while preserving core stack, alignment, overlays, and essential philosophy blocks.

### Portable Vault Export
Reloadable logic spine intended for cloning, restoration, or future vault loading.
