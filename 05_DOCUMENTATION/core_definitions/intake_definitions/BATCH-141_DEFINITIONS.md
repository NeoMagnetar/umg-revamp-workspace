# BATCH-141 — Definitions Extract

## Terminology Scope Note
These definitions are extracted from a derived modular-builder summary.
They remain Stage 1 evidence until explicitly promoted elsewhere.

## Core Builder Terms
**Block Anatomy**  
Structured field set describing a modular build unit through id, type, category, dependencies, contains, merge targets, outputs, snap targets, and description.

**Primary Block**  
Master controller block for a full website/app pipeline.

**Merge Block**  
Composition unit defining what gets fused and in what order.

**Snap / Snap-To**  
Logical fit relationship indicating where a block belongs in the larger build structure.

**Function Block**  
Executable behavior unit such as install/dev/build/start or backend/API action.

**File Block**  
Concrete file artifact such as HTML, CSS, config, or server file.

**Wrapper Block**  
Packaging or shell layer such as PWA/mobile wrapper.

## Packaging Terms
**Logic Bundle**  
Zip or bundle containing reusable `.canto` logic blocks and protocol instructions.

**Code Bundle**  
Zip or bundle containing executable project files for deployment/runtime use.

**Build + Inquiry Protocol**  
Assistant behavior in which the system returns both a build artifact/spec and a clarifying inquiry about implied or missing components.
