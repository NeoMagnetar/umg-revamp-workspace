## High-Signal Definitions

### Cognitive Systems Framework (CSF)
- Existing parent category
- Marked in the source as non-novel
- Serves as a broader classification rather than a UMG-specific construct

### Universal Modular Generation (UMG)
- Framework-level construct
- Defined here as the modular generation of cognitive processes
- Explicitly separated from execution
- Positioned inside CSL

### Cognitive Specification Layer (CSL)
- Non-executing jurisdiction / layer for cognition
- Upstream of runtime
- Defines where cognition is specified and resolved

### Modular Operating Language of Thought (MOLT)
- Language for cognition
- Positioned inside CSL and used by UMG
- Defined here by negative distinction as not a programming language, prompt language, or policy language

### Cognitive Intermediate Representation (CIR)
- Compiled cognition
- IR for cognition rather than code or prompts

### RuntimeSpec
- Enforceable cognitive constraint artifact
- Sits at the boundary between control and execution

### Decision Trace
- Deterministic cognitive resolution record
- Audit-layer artifact

### Governance Layer
- Authority + priority resolution for cognition
- Positioned above CSL and compilation
- Framed as governance-before-execution

### Dynamic Context Engineering Protocol (DCEP)
- Execution-time context assembly protocol
- Positioned at the execution boundary
- Framed as guided inference without planning; protocolized context rather than prompts

### System 1+ (S1+)
- S1 + DCEP
- Guided reactive reasoning without deliberation

### System 2′ (S2′)
- Constructed, externalized deliberation
- Artifact-governed reasoning

### Cognitive Control Plane
- Architectural plane between applications and models
- Framed as governance of cognition itself like an OS / compiler pipeline layer
