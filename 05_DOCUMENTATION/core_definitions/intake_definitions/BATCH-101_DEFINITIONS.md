# BATCH-101_DEFINITIONS

## Scope Note
These definitions are extracted from a derived source summary. They are evidence candidates, not canon.

## Terminology Extract

### composable AI thinking
- Language used to describe UMG as modular, swappable, and structurally organized reasoning rather than monolithic prompting.

### structure of thought
- Phrase used to describe UMG’s purpose as organizing reasoning form, not only generating outputs.

### interface for AGI-level cognition
- Positioning language suggesting UMG functions as a structured layer for high-level reasoning and modular orchestration.

## Role Language

### Primary
- Core intent layer described as “what you want.”
- Treated as the main directive driving structured reasoning.

### Merge
- Context/data layer described as “what to include.”
- Treated as the mechanism for injecting information, external data, or feedback.

### Blueprint
- Style/strategy layer described as “how it should feel.”
- Treated as the rendering or strategy surface shaping the external output.

## Architecture Language

### modular cognitive decomposition
- Reasoning model where intent, context, and strategy are separated into distinct modules or blocks.

### plug-and-play blocks
- Block behavior where modules can be independently edited, swapped, and recomposed.

### hybrid reasoning/control architecture
- Layered model where UMG handles higher-level reasoning while MPC handles lower-level control or execution.

## Runtime / Compiler Language

### structured prompt composition
- Implied execution model where multiple modules are combined into one organized prompt or instruction structure.

### iterative refinement loop
- Runtime-like behavior where outputs are revised over multiple modular reasoning cycles.

### concatenated modular instructions
- Phrase implying a compile/runtime boundary where separate reasoning modules are merged into a single executable prompt structure.

## Rendering / Output Language

### content versus style separation
- Distinction where Primary and Merge contribute content logic while Blueprint contributes rendering logic, tone, or approach.

## Governance / Safety Language

### modular constraint injection
- Indirect safety/control idea where rules or guardrails can be inserted through modular structure rather than only global system rules.

### tone/ethical shaping
- Soft governance function implied through Blueprint or Merge without a fully formal governance layer in this source.
