# BATCH-202 — Definitions Extract

## Terminology
### Progressive Disclosure
Human-interface simplification technique for staged reveal of complexity.

### UMG
Modular architecture for cognition, governance, control, and runtime organization.

### Skill / Plugin
Deployment container through which UMG becomes active inside an agent runtime.

### ClawHub
Registry/distribution layer for finding, installing, and updating skills/plugins.

## MOLT / Context Language
### Trigger
Question or activation request such as comparing an external concept to UMG.

### Directive
Requested comparison or interpretation frame.

### Instruction
Operational framing for how the comparison should be made.

### Subject
The compared concept domain.

### Primary
Core question or current focus.

### Philosophy
Underlying lens for distinguishing architecture from presentation.

### Blueprint
Surface explanation or communication framing.

## Boundary Language
### Surface Behavior
UI/operator-facing reveal strategy.

### Underlying Ontology and Control Grammar
Internal structure governing cognition, activation, and audit.

## Definitional Candidates
- Progressive disclosure could be a UI strategy inside UMG, but it is not equivalent to UMG.
- UMG is internal architecture and governance; progressive disclosure is optional interface strategy.
- Progressive disclosure answers how much the user should see right now; UMG answers how cognition and authority should be partitioned, activated, and audited.
