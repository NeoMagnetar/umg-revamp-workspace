# BATCH-121_DEFINITIONS

## Rule-Language Extract
- **UMG governs cognition, not behavior.**
- **Cognition must be explicit.**
- **If something matters to reasoning, it must exist as a named object.**
- **If it is not written down, it does not exist.**
- **If it exists, it must be inspectable.**
- **Blocks are meaning, not action.**
- **A block answers exactly one cognitive question.**
- **A block belongs to exactly one type.**
- **Trigger:** eligibility gate only; adds no meaning or authority.
- **Priority:** deterministic local conflict resolver only.
- **Governance:** supreme participation control.
- **OFF:** absolute exclusion mechanism.
- **Determinism:** identical blocks, structure, governance, triggers, and priorities must yield the same result.
- **Traceability:** if no trace exists, the outcome is invalid.

## Role / Question Mapping
- Why -> Purpose
- About what -> Subject
- How (strategy) -> Directive
- Under what rules -> Instruction
- Under what values -> Philosophy
- In what form -> Blueprint
- When eligible -> Trigger

## Separation Language
- meaning != action
- eligibility != authority
- governance != strategy
- specification != execution
- traceability != optional metadata

## Caution
- Merge is only lightly referenced here.
- Blueprint is named but not deeply elaborated.
- OFF is asserted as absolute without full representation details in this batch.
