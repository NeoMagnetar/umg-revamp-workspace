# BATCH-125 — Definitions Extract

## Terminology Scope Note
These terms are extracted from a derived implementation/recovery summary.
Some are clearly canonical or already familiar UMG terms; others appear implementation-local or provisional and should remain evidence until explicitly normalized.

## Canonical / Near-Canonical Terms Present
**MOLT Stack (seven-role order)**  
Trigger → Directive → Instruction → Subject → Primary → Philosophy → Blueprint.

**PriorityOrder**  
Integer priority field used in validation and inspector workflow, with lower value representing higher priority in the described implementation context.

**NeoBlock**  
Compressed structure derived from a full MOLT spine, preserving source block identity and later including MOLT snapshot data.

**NeoStack**  
Composition artifact formed from multiple NeoBlocks and treated as a distinct graph/render artifact before sleeve creation.

**Sleeve**  
Highest-level artifact in the tutorial flow; stable container identity for the resulting composed structure.

**RuntimeSpec**  
Rendered active-structure output used to mirror creator output and separate active structure from raw/internal state.

**Trace**  
Recorded explanation/output surface for how tutorial or compile state was derived.

## Provisional / Implementation-Local Terms
**Baseline**  
Known-good stable state preserved before additional mutation.

**Baseline Sleeve**  
Candidate phrase for a locked, known-good sleeve state used as a mutation anchor.

**Trace Gate**  
Candidate process term for requiring validation/inspection before additional edits or mutation.

**Spine / Spine Blocks**  
One-block-per-role completeness model for the seven MOLT roles, with extra blocks allowed but not needed for basic completion.

**Dominant Snapshot Rule**  
Simplified composition rule in which one NeoBlock snapshot overrides another during merge-like behavior.

**Fact-Finding Only Mode**  
Agent operating restriction focused on observation and analysis without direct code mutation.

## Boundary Language Present
internal state ≠ rendered output  
partial implementation ≠ canonical semantics  
bounded agent behavior ≠ implicit trust  
grouping/composition mechanic ≠ fully formalized merge semantics
