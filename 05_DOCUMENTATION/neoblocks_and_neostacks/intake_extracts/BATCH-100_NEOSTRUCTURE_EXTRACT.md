# BATCH-100_NEOSTRUCTURE_EXTRACT

## Scope
NeoBlock / NeoStack / graph/editor structure signals extracted from BATCH-100.

## Stack / Editor Construction Signal
The source reports a visual editor with:
- drag-and-drop stack assembly
- 5-block starter stacks
- real-time Cantocore preview
- WebSocket hot reload

This provides one of the clearest first-release user-facing neostructure surfaces in the intake set.

## Graph Runtime Signal
The batch does not describe stacks as only ordered lists. It explicitly frames runtime execution as graph-based.

This pressures neostructure work to clarify:
- relationship between stack view and graph view
- whether the editor shows linear stacks, DAG-like relations, or both
- how user-facing assembly maps onto runtime graph normalization
- how preview/edit operations preserve graph integrity

## Starter Composition Signal
Starter blocks and starter stacks imply a curated first-release composition model:
- small, legible assemblies first
- reusable named blocks
- visible structure before large-scale automation
- editor-mediated composition as part of adoption strategy

## Bundle / Product Signal
The source links structure to launch surfaces:
- visual editor
- demos
- SaaS release
- deployment pipeline

That suggests the first neostructure exposure is not only internal documentation, but productized interaction with block/stack/graph structure.

## Structural Tensions
- “5-block stack” may be a UX simplification rather than the full runtime limit or canonical structural rule
- graph execution and stack assembly may represent different views of the same underlying system
- Overlay and Merge semantics may cut across pure stack ordering
- hot reload and real-time preview add product pressure before core semantics are fully stabilized

## High-Value Documentation Targets
- define stack view versus graph view explicitly
- document editor-facing composition constraints
- clarify how starter stacks map into runtime graphs
- separate MVP UX simplifications from long-term canonical structure
- define what parts of neostructure are product affordances versus semantic core
