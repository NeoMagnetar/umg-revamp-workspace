# BATCH-095_NEOSTRUCTURE_EXTRACT

## Scope
NeoBlock / NeoStack / dynamic-composition signals extracted from BATCH-095.

## Candidate Specialized NeoBlocks
### Geometry encoder
- candidate specialized block for spatial or shape-structured input
- not a full cognition system by itself
- strongest value is as a modular specialist

### Language reasoning model
- candidate reasoning/output block
- handles generalized language interpretation and response generation

### Adapter / Fusion block
- candidate bridge block enabling interoperability between specialized representations and reasoning/output layers
- structurally important because it mediates between otherwise separate modules

### LoRA-like micro-blocks
- candidate local modification units that alter behavior without full retraining
- may be better treated as modifiers than as full independent blocks

## Candidate Stack / Graph Structure
The source implies a compositional architecture built from:
- geometry processing layer
- language reasoning layer
- adapter / fusion layer

The source also implies that stack composition may be task-dependent rather than fixed.

## Structural Importance
This batch pressures UMG to define whether:
- specialized blocks are a first-class block category or only implementation examples
- bridge blocks deserve their own architectural status
- dynamic stack assembly is part of UMG semantics or only future-facing design exploration
- modifier units like LoRA belong in the same structural family as blocks or in a separate tuning layer

## Open Structural Tensions
- relation between stack and graph remains underspecified
- boundary between block, layer, and module is not stabilized
- dynamic assembly may exceed current UMG semantics if adopted too literally
- bridge behavior is central but not yet formally named in existing UMG structures
