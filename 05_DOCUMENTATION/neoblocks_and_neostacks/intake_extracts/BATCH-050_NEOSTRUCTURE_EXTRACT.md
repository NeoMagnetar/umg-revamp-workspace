# BATCH-050 — Neostructure Extract

## NeoStack Findings
- NeoStacks were firmly treated as domain containers.
- Example domain families implied by source:
  - core identity
  - future vector / time depth
  - strategic engine
  - modular integration
  - risk engine
  - expression / surface
  - runtime agency loop domains

## NeoBlock Findings
- NeoBlocks are the correct grouped-intent layer between domain containers and atomic MOLT primitives.
- They are the likely boundary for:
  - semantic cohesion
  - mutation policy
  - auditability
  - release/version control

## Representation Model Tension
- The batch strongly supports a hybrid spatial+weight representation for advanced sleeve semantics.
- Unresolved: whether that hybrid becomes canonical or remains advanced overlay while linear MOLT remains interchange format.

## Candidate Neostructure Decisions
1. Keep linear MOLT as default interchange/wire format.
2. Allow advanced spatial+weight overlay for execution semantics and visualization.
3. Keep NeoStacks/NeoBlocks explicit and compiler-checkable in both representations.
4. Preserve external governance modeling above sleeve structure.
