# BATCH-012 — NeoStructure Extract

## NeoBlock Signal
- NeoBlock is the compiled active view inside a stack.
- One Primary per stack remains explicit.
- Instructions may layer.
- Bundles affect activation and ordering.

## NeoStack Signal
- NeoStacks are explicit modes of cognition/work.
- Activation is explicit and ON/OFF-aware.
- Workmode examples in the guide include Compiler, GitHub, Studio, and WarMode.
- The mode catalog expands this into a much larger sleeve-mode library.

## Ordered Runtime NeoStructure
- Bounded-agency runtime uses ordered NeoStacks:
  - `NSTK.GOAL.GENESIS.v1`
  - `NSTK.GOAL.SELECTION.v1`
  - `NSTK.SELF_IMPROVEMENT.RUNTIME.v1`
- Each stack is composed of named atoms with role-like functions.

## Intake Value
This batch is strong evidence that UMG NeoStructure is becoming more operational and typed, not just descriptive. It is especially useful for later normalization work between conceptual NeoStacks and runtime-execution NeoStacks.

## Caution
- Mode catalog breadth exceeds what is normalized in the visible batch.
- Runtime stack details are strong, but the overall runtime spec is still DRAFT.
