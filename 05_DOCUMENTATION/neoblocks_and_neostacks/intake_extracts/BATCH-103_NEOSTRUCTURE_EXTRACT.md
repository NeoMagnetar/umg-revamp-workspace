## Scope
NeoBlock / NeoStack / branching-architecture signals extracted from BATCH-103.

## Stack Signal
The source describes UMG as a visual or hierarchical stack that gives humans upfront control over reasoning structure.

That baseline matters because the packet does not reject stacks. Instead, it asks how stacks might support more adaptive behavior.

## Branching / Hybrid Signal
The strongest neostructure pressure comes from:
- scenario branching
- parallel UMG agents
- result aggregation
- simulation-assisted comparison across variants

This suggests possible movement from simple ordered stacks toward:
- stack-plus-branch structures
- graph-like execution layers
- parallel comparison ensembles

## Authority Signal
Even with adaptive features, the source repeatedly values:
- human legibility
- explicit alignment surfaces
- designer control before runtime

This suggests future advanced structures would still need clear visibility and bounded authority.

## Structural Tensions
- simulation-style branching may conflict with lean linear-stack models
- branch visibility requirements may force new UI and trace surfaces
- aggregator logic may require graph or ensemble semantics rather than only one-stack execution
- compute cost may limit whether such structures are baseline or optional

## High-Value Documentation Targets
- define whether branch/simulation structures are optional advanced overlays on top of standard stacks
- document hybrid parallel-agent ensembles separately from baseline single-stack UMG
- preserve legibility requirements when introducing graph-like or branching execution
- distinguish simple stack architecture from simulation-assisted multi-path reasoning architecture
