# BATCH-176 — Sleeve Extract

## Scope
Sleeve identity and compilation-boundary signals extracted from the canon and compiler source.

## Direct Sleeve Signal
The source repeatedly defines Sleeve as:
- a lifeless, static container of potential cognition
- containing blocks, domains, snaps, defaults, and metadata
- not an agent
- not runtime execution
- not memory or runtime state

## Compilation Meaning
Sleeves are what a system could be under different contexts.
They become active only when a snap is selected and the compiler emits RuntimeSpec plus Trace.

## Preservation / Evolution Signal
Older cognition artifacts should be preserved and deactivated with Off rather than silently overwritten.
This ties sleeve stability to versioning and canon continuity.

## Structural Doctrine
- sleeve = potential cognition
- runtime = external execution
- sleeve contents remain declarative until compilation
- preservation is favored over destructive mutation
