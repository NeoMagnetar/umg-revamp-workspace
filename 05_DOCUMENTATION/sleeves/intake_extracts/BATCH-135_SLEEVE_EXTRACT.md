# BATCH-135 — Sleeve Extract

## Scope
Sleeve and mounted-skill architecture material extracted from the source.

## Agnostic Sleeve Pattern
A more agnostic bot sleeve was defined as a reusable base chassis for deployable systems.
Suggested base domains included:
- Identity
- Communication
- Instruction
- Philosophy
- Knowledge
- Reasoning
- Personality
- UserModel
- Behavior
- Constraints
- Operations
- Escalation
- Evaluation
- Trace
- Governance

## Mounted Skillpack Pattern
A specialized NeoStack or NeoSleeve can be mounted into the agnostic sleeve as a skill cartridge or skillpack.
This lets the base chassis remain stable while specialized work systems provide domain capability.

## Person Sleeve Pattern
A separate reusable person-sleeve tree was also proposed with domains such as:
- Identity
- Personality
- Philosophy
- Beliefs
- Behavior
- Skills
- Constraints
- Relationships
- Environment
- Conflict
- NarrativeFunction
- Trace

## Sleeve-Level Doctrine
- Use the agnostic sleeve as chassis.
- Mount specialized sleeves or NeoStacks rather than rewriting the whole base system.
- Constrain mounted systems with trace, escalation, governance, and explicit contracts.
- Preserve separation between base cognitive structure and specialized skill domains.
