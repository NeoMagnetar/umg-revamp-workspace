# BATCH-020 — Sleeve Extract

## Sleeve Role in This Batch
Sleeves are treated as first-class stored assets in the private homebase repo and as explicit runtime identity controls rather than vague persona files.

## Strongest Sleeve Signals
- sleeves belong in a dedicated asset layer
- one agent should be able to store and switch its own sleeves
- runtime should track the active sleeve explicitly
- sleeve switching belongs in the homebase workflow, not in ad hoc chat state

## Runtime Placement
Relevant sleeve-facing surfaces signaled by the batch:
- `sleeves/`
- `runtime/active-sleeve.json`
- compiler-aware reconfiguration
- workspace-skill behavior contract

## Caution
The batch intentionally deprioritizes broad governance design. Sleeve handling here is practical and private-first, not full public ecosystem canon.
