# BATCH-195 - Sleeve Extract

## Scope
Sleeve identity and compiler-boundary signals extracted from the CSL/compiler-foundation source.

## Direct Signal
Sleeve was defined as a complete cognitive loadout/specification containing:
- blocks
- modules/stacks
- snaps
- defaults

Sleeve is repeatedly treated as:
- agent-ready
- not executable
- distinct from Agent

## Structural Meaning
A sleeve is the full cognitive loadout that the compiler interprets into RuntimeSpec and Trace.
The source keeps Sleeve on the specification side of the boundary and Agent on the execution side.

## Useful Value
This batch strongly supports Sleeve as:
- static cognitive specification
- swappable loadout
- compiler input boundary

## Caution
No formal resleeving protocol was finalized, though sleeves are clearly treated as swappable cognitive loadouts.
