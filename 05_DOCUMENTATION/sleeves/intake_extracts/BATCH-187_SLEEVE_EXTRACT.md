# BATCH-187 - Sleeve Extract

## Scope
Sleeve identity and compilation-boundary signals extracted from the canon-formalization source.

## Direct Sleeve Signal
Sleeve was defined as the highest-level container of potential cognition.

A sleeve contains:
- blocks
- domains
- snaps
- defaults
- metadata

Sleeve is explicitly:
- lifeless
- distinct from an agent
- distinct from runtime state

## Compilation Meaning
A sleeve becomes meaningful only when a snap is selected and the compiler emits RuntimeSpec plus Trace.

## Structural Doctrine
- sleeve = potential cognition
- runtime = external execution
- old blocks should be preserved, disabled, or replaced through versioning rather than silently mutated
