# BATCH-036 — Sleeve Extract

## Sleeve-Relevant Signal
The batch repeatedly treats the sleeve as a runtime environment / session configuration layer rather than as the whole system. This is useful because it prevents sleeve inflation: governance stays above, stacks and blocks stay below, and execution output stays downstream.

## Sleeve Framing Candidates
- sleeve as active runtime environment
- sleeve as user environment / session context in the OS analogy
- sleeve as mutable operating mode under governance, not as the governance anchor itself

## Why This Matters
This framing helps keep sleeves from absorbing responsibilities that belong elsewhere:
- governance should not collapse into sleeve personality
- stack execution should not collapse into sleeve prose
- output rendering should not collapse into sleeve identity
- runtime/controller enforcement should not be confused with sleeve text

## Practical Value
Useful for future sleeve documentation because it sharpens boundaries between:
- governance
- sleeve
- NeoStacks
- NeoBlocks
- execution output

## Status
Evidence only. Treat as cross-batch sleeve-boundary support, not as standalone sleeve canon.
