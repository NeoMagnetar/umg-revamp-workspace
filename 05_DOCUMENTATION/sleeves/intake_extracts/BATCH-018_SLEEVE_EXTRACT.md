# BATCH-018 — Sleeve Extract

## Sleeve Role in This Batch
Sleeves are treated as the current runtime identity layer and as a distinct step in the runtime pipeline, not something collapsed into compiler or renderer behavior.

## Strongest Sleeve Signals
- sleeve selection/loading should be its own stage
- sleeves contain defaults, constraints, active stacks, and governance-adjacent posture
- resleeving should occur before rendering so output reports resolved state
- sleeves are not cosmetic personas; they are runtime identity containers

## ClawHub / Runtime Placement
Relevant sleeve-facing stages proposed in the batch:
- UMG Sleeve Selector / Loader
- UMG Reconfiguration Engine
- UMG Compiler / Validator
- UMG Output Envelope Renderer

## Caution
Governance is repeatedly acknowledged as not yet fully solved, so sleeve/governance boundaries remain evidence-level rather than canon-locked in this batch.
