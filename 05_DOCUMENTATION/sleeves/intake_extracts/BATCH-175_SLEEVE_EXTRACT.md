# BATCH-175 — Sleeve Extract

## Scope
Sleeve identity and compilation-boundary signals extracted from the canon-building source.

## Direct Sleeve Signal
The source repeatedly defines Sleeve as:
- a lifeless container of all possible cognition
- holding blocks, snaps, defaults, domains, and metadata
- static and declarative
- not an agent
- not runtime state
- not execution

## Compilation Meaning
A Sleeve becomes meaningful only when compiled with a selected snap into RuntimeSpec plus Trace.
This strongly preserves sleeve as authored potential rather than active behavior.

## Structural Doctrine
- sleeve = potential cognition
- agent/runtime = external execution layer
- canon discourages silent mutation of active meaning
- old blocks may become Off and remain preserved instead of being erased
